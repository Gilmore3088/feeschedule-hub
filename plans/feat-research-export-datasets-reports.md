# Research Export: Datasets & Executive Reports

## Overview

The Fee Analyst and Custom Query agents produce rich data-driven analysis in the chat — tables, comparisons, benchmarks — but there's no way to export that output. Users need two export paths:

1. **Dataset export (CSV)** — raw data tables from agent analysis for spreadsheets/BI tools
2. **Executive report (PDF/HTML)** — polished, branded output for presentations and stakeholders

## What Exists Today

- Fee Analyst agent produces markdown tables, bullet points, and analysis
- Custom Query agent does multi-step analysis with tool calls
- Content Writer generates long-form articles (saveable as drafts)
- Market page has CSV export via `exportMarketCsv()` server action
- No export from chat interface

## Phase 1: Export Buttons in Chat UI

### 1a. "Export as CSV" button

When an assistant message contains a markdown table, show an "Export CSV" button that:
- Parses the markdown table into rows/columns
- Downloads as `.csv` file with timestamped filename
- Works client-side (no server action needed)

### 1b. "Export as Report" button

For any substantive assistant response (>200 chars), show an "Export Report" button that:
- Wraps the markdown in a branded HTML template
- Includes Bank Fee Index header, date, agent attribution
- Opens browser print dialog (Print to PDF)
- Clean print stylesheet: no chat chrome, just the content

### CSV Parser

```typescript
function markdownTableToCsv(markdown: string): string {
  const lines = markdown.split("\n").filter(l => l.trim().startsWith("|"));
  if (lines.length < 2) return "";

  return lines
    .filter(l => !l.match(/^\|[\s-:|]+\|$/)) // skip separator row
    .map(line =>
      line.split("|").filter(Boolean).map(cell =>
        `"${cell.trim().replace(/"/g, '""')}"`
      ).join(",")
    )
    .join("\n");
}
```

## Phase 2: Dataset Builder Tool

### 2a. New agent tool: `buildDataset`

Add a tool to the Fee Analyst that explicitly builds structured datasets:

```typescript

    columns: z.array(z.string()).describe("Column names"),

      states: z.array(z.string()).optional(),
      charter: z.enum(["bank", "credit_union"]).optional(),
      tier: z.string().optional(),
    }).optional(),
    metrics: z.array(z.enum(["median", "p25", "p75", "min", "max", "count"])),
    groupBy: z.enum(["category", "state", "district", "charter", "tier"]),
  }),
  execute: async ({ filters, metrics, groupBy }) => {
    // Query DB, return structured rows
    return { columns: [...], rows: [...], row_count: N };

```

This lets users say: "Build me a dataset of overdraft and NSF medians by state" and get a clean, exportable table.
