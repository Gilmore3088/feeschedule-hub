# Premium Agent Access & Branded Output Templates

## Overview

Two related needs:

1. **Premium user tier** — Research agents (Fee Analyst, Content Writer) are currently admin-only. Need a "premium" public user tier so paying customers can access agents without admin accounts.
2. **Branded output templates** — When agents produce reports, exports, or articles, the output should use professional Fee Insight / Bank Fee Index branded templates.

## Current State

| Capability | Today | Needed |
|------------|-------|--------|
| Auth roles | viewer, analyst, admin (internal only) | + premium (external paying users) |
| Public agent | "Ask the Data" (free, rate-limited) | Keep as-is |
| Fee Analyst | Admin login required | Available to premium users |
| Content Writer | Admin login required | Admin only (internal tool) |
| Custom Query | Admin login required | Admin only |
| Export Report | Generic HTML template | Branded Fee Insight template |
| Export CSV | Plain CSV download | Add branded header row |
| Article pages | Basic markdown rendering | Branded layout with logo |

## Phase 1: Premium User Tier

### 1a. Add `premium` role to auth system

**File:** `src/lib/auth.ts`

```typescript
role: "viewer" | "analyst" | "admin" | "premium";
```

Premium users:
- Can access Fee Analyst agent (currently analyst+)
- Cannot access Content Writer or Custom Query (admin only)
- Cannot access admin pages (review, pipeline, etc.)
- Have their own rate limits (e.g., 20 queries/day)
- Get branded output templates

### 1b. Premium login flow

Options (pick one):
- **Option A: Magic link** — email-based, no password. User enters email, gets a login link. Simple, secure.
- **Option B: API key** — premium users get an API key they include in requests. Good for programmatic access.
- **Option C: Session-based** — traditional email/password like admin. Most familiar.

**Recommendation:** Start with Option C (session-based) since the auth infrastructure already exists. Add API key access later for programmatic use.

### 1c. Premium agent page

Create `/pro/research` (or `/premium/research`) — a dedicated page for premium users:
- Fee Analyst agent with full tool access
- Branded output with export buttons
- Usage dashboard (queries remaining today)
- No access to admin nav, review queue, pipeline, etc.

### 1d. Rate limiting for premium

**File:** `src/lib/research/rate-limit.ts`

```typescript
const PREMIUM_LIMITS = {
  queries_per_day: 20,
  queries_per_minute: 5,
  max_tokens_per_query: 4096,
};
```
