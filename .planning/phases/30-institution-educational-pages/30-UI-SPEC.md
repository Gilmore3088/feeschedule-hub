## Component Inventory

New components introduced by this phase and their visual contract.

### 1. FeeCallout

**Purpose:** Educational "why does this matter?" block rendered below each spotlight/core fee row in the fee table.

**Trigger:** Rendered for fee categories that have a static explainer entry in the category-explainers data file. Only shown for categories where index data is also available (to supply the quantified comparison).

**Visual anatomy:**
- Container: `mt-3 rounded-lg border border-[#E8DFD1]/60 bg-[#FAF7F2]/70 px-4 py-3`
- No icon or decorative mark — copy alone carries the meaning
- First sentence (static explainer): `text-[13px] leading-relaxed text-[#5A5347]`
- Second sentence (live quantification): `text-[13px] leading-relaxed text-[#1A1815] font-medium` — bold to distinguish from static copy
- Pattern: "{Static explainer for what this fee is.} {Institution name} charges {$X}, compared to the national median of {$Y} for {charter type} institutions."

**Tone rule (from D-03):** Human, direct, no jargon. Real dollar amounts. "Your bank charged you $35 for a $4 coffee" energy — not "This fee represents a significant cost burden."

**Empty/missing state:** No callout rendered — fee row shows normally. Do not render placeholder or loading state.

### 2. PercentileBadge

**Purpose:** Inline indicator showing where this institution's fee ranks against same-charter, same-tier peers. Appended to existing PositionBar component — not a standalone replacement.

**Decision (D-04):** Augment PositionBar with a text label appended after the delta percentage column. Keeps all positioning data in one scannable row.

**Visual anatomy:**
- Position: fifth column in PositionBar row, right-aligned, 60px wide
- Text: `text-[10px] tabular-nums font-medium`
- Color: emerald-600 if below 50th percentile (below median = consumer-favorable), red-600 if above 50th percentile, #A09788 if exactly at 50th
- Format: "top 28%" (means lower than 72% of peers, i.e., fee is lower = better for consumer)
- Tooltip on hover (title attribute): "Higher than {N}% of {charter} institutions in the same asset tier"
- "top X%" framing: "top" refers to best-for-consumer (lowest fee), not highest fee
  - If fee is at 30th percentile (below 70% of peers) → "top 30%" in emerald
  - If fee is at 80th percentile (above 80% of peers) → "bottom 20%" in red

### 3. FeeDistributionSection

**Purpose:** Dedicated section showing histogram for spotlight fees (up to 6 categories). Shows where THIS institution sits in the national distribution.

**Decision (D-05, D-06, D-07):** One section, not per-row expandable. Reuses FeeHistogram component. Institution position is a highlighted vertical line (or colored bar) distinct from the gray national distribution.

**Access gate (D-08 — Claude's Discretion):** Section is FREE (no pro gate). Rationale: this is the strongest visual differentiator from NerdWallet/Bankrate. Putting it behind a gate removes the hook. The B2B report CTAs immediately below are pro-gated — the distribution is the reason to want the reports.

**Layout:**
- Section wrapper: `mt-12` below the CompetitiveScorecard section
- Section heading: `text-[16px] font-medium text-[#1A1815]` in Newsreader serif — "Fee Distribution"
- Section subtitle: `mt-1 text-[13px] text-[#7A7062]` — "Where {institution name}'s fees sit in the national distribution."
- Chart grid: `mt-5 grid grid-cols-1 gap-6 sm:grid-cols-2 lg:grid-cols-3`
- Each chart card: `rounded-xl border border-[#E8DFD1]/80 bg-white/70 px-4 py-4`
- Chart card label: `text-[11px] font-bold uppercase tracking-[0.1em] text-[#A09788]` — category display name
- Chart card institution marker label: `text-[10px] text-[#5A5347]` — "This bank: $35"

**Institution position marker in histogram:**
- A vertical ReferenceLine at institution's fee amount, color: `#C44B2E` (accent), strokeWidth: 2, strokeDasharray: "3 3"
- Tooltip label on that ReferenceLine: "{Institution name}: {$amount}"
- The existing median ReferenceLine stays as-is (uses #A09788)

**Categories shown (up to 6 if data exists):** overdraft, nsf, monthly_maintenance, atm_non_network, wire_domestic_outgoing, card_foreign_txn — in this order. If the institution has no approved fee for a category, that histogram is omitted (do not render empty charts).

**Empty state (institution has fewer than 2 of these 6 fees):** Section is omitted entirely. Do not render the section heading.

### 4. IntelligenceSection (B2B Links — D-09, D-10, D-11)

**Purpose:** Pro-only section linking to pre-built peer/district/tier reports and offering Hamilton CTAs.

**Layout position:** Bottom of page, after fee table, before the Data Sources methodology footer. Replaces existing "Report Card + Links" pill row (absorbs those links into the new section).

**Access gate:** Full section content is pro-gated via UpgradeGate. Consumers see UpgradeGate compact variant with message: "Unlock competitive intelligence — see how your peer group compares."

**Pro layout:**
- Section wrapper: `mt-10`
- Section heading: `text-[16px] font-medium text-[#1A1815]` in Newsreader serif — "Intelligence & Reports"
- Section subtitle: `mt-1 text-[13px] text-[#7A7062]` — "Reports and analysis for this institution's peer group."
- Two-column on desktop (`grid grid-cols-1 gap-4 sm:grid-cols-2`), stacked on mobile

**Left column — Related Reports:**
- Heading: `text-[10px] font-bold uppercase tracking-[0.1em] text-[#A09788]` — "Related Reports"
- Report links (if any): each is a Link styled as a pill matching existing report pill pattern — `rounded-full border border-[#E8DFD1] px-4 py-1.5 text-[12px] font-medium text-[#5A5347] hover:border-[#C44B2E]/30 hover:text-[#C44B2E] transition-colors`
- Empty state (no matching reports): `text-[13px] text-[#A09788]` — "No peer reports published yet for this segment."

**Right column — Hamilton CTAs:**
- Heading: `text-[10px] font-bold uppercase tracking-[0.1em] text-[#A09788]` — "Ask Hamilton"
- Two CTA links as rounded-full pills in accent style (D-09):
  - "Generate a competitive brief" — links to `/pro/research?prompt=competitive-brief&instId={id}`
  - "Ask about this institution" — links to `/pro/research?prompt=institution&instId={id}`
- CTA pill style: `inline-flex items-center gap-1.5 rounded-full border border-[#C44B2E]/30 bg-[#C44B2E]/5 px-4 py-1.5 text-[12px] font-semibold text-[#C44B2E] hover:bg-[#C44B2E]/10 transition-colors`
- Existing "Fee Report Card", state report, and district report pills move here (below Hamilton CTAs, styled as secondary pills)

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| FeeCallout visibility | Always visible below qualifying fee rows — no expand/collapse toggle |
| PercentileBadge tooltip | Native `title` attribute (no JS tooltip library) — accessible, zero-weight |

## Data Explainer Map Contract

The 49-category explainer strings live in a TypeScript `const` map (not JSON, not a database) at `src/lib/fee-explainers.ts`. Each entry is:

```ts
type FeeExplainer = {
  category: string;       // fee_category slug (e.g. "overdraft")
  explainer: string;      // 1 static sentence, plain language, no dollar amounts
};
```

Only spotlight + core categories (15 total) require explainers at launch. Extended/comprehensive categories show no callout if their explainer is absent.

Spotlight categories requiring explainers (must be authored at implementation):
- overdraft, nsf, monthly_maintenance, atm_non_network, card_foreign_txn, wire_domestic_outgoing

Core categories requiring explainers (9 remaining):
- savings_excess_withdrawal, wire_domestic_incoming, stop_payment, cashier_check, money_order, account_closure, paper_statement, dormancy, safe_deposit_box
