## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badges |
| sm | 8px | Button icon gap, tight label spacing |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section inner padding, card gap |
| xl | 32px | Between hero elements, card grid gap |
| 2xl | 48px | Section vertical padding (mobile) |
| 3xl | 64px | Section vertical padding (desktop: py-16 = 64px, lg:py-20 = 80px) |

Exceptions:
- Hero search bar touch target: minimum 48px height (py-3 = 12px * 2 + 24px line = 48px total)
- Value prop card touch target: minimum 44px for any clickable element within card
- CTA button: py-3.5 = 56px height (rounded-full pill shape) — deliberate larger than 44px base

Source: Established consumer page pattern at `mx-auto max-w-6xl px-6 py-16 lg:py-20` (CONTEXT.md code_context)

## Typography

Permitted sizes: 12, 14, 28, 40 (4 sizes)
Permitted weights: 400 (regular), 700 (bold) (2 weights)

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display (h1 hero) | clamp(28px, 5vw, 40px) | 400 (italic for emphasis via `<em>`) | 1.08 | Newsreader serif |
| Heading (h2 sections) | 28px | 400 | 1.15 | Newsreader serif |
| Body / UI text / label | 14px | 400 | 1.5–1.6 (leading-relaxed) | Geist Sans |
| Data / stat number | 28px | 700 | 1.0 | Geist Sans + tabular-nums |
| Caption / meta | 12px | 400 uppercase (tracking-[0.1em]) | 1.4 | Geist Sans |

Notes on consolidation:
- Body (previously 15–17px step-up) unified at 14px. Desktop step-up removed.
- Stat numbers (previously 32px) moved to 28px — shares the heading size slot, differentiated by weight 700 and tabular-nums.
- Captions (previously 11–12px semibold) unified at 12px weight 400. Uppercase + tracking-[0.1em] provides visual differentiation without a separate weight.
- Card titles (previously 20px semibold) move to 14px weight 400 with uppercase tracking to maintain hierarchy within the body size.
- H2 sections (previously 24px mobile / 28px desktop) unified at 28px flat.

Source: CONTEXT.md established patterns + consumer/page.tsx hero markup confirmed

## Page Sections & Component Inventory

This landing page has exactly 3 sections (D-04 from CONTEXT.md). Each section's components are specified below.

### Section 1: Hero

**Layout:** Full-width within `mx-auto max-w-6xl px-6 py-16 lg:py-20`. Content constrained to `max-w-2xl`.

**Components:**
- `<h1>`: Display size (clamp 28–40px), Newsreader, "What is your bank *really* charging you?" with `<em>` italic on "really" at weight 400
- Supporting text: 14px, #6B6355, "Compare fees across {stats.total_institutions.toLocaleString()} banks and credit unions."
- `InstitutionSearchBar` (extracted from `src/app/(public)/institutions/search-bar.tsx`): Embedded directly, no auth gate, renders server-side wrapper client component. Max-width: `max-w-xl` on desktop, full-width on mobile.
- Subtle gradient accent: `bg-gradient-to-l from-[#C44B2E]/[0.03]` positioned top-right (existing pattern from consumer/page.tsx)

**Search bar states:**
- Default: border #E8DFD1, bg #FFFDF9, placeholder "Search your bank or credit union..."
- Focus: ring-2 ring-[#C44B2E], border transparent
- Loading: spinner top-right inside input (border-t-[#C44B2E], animate-spin)
- Results dropdown: bg #FFFDF9, border #E8DFD1, rounded-xl, shadow-lg, z-50
- Each result row: institution name (14px, #1A1815), location + charter type + fee count (12px, #7A7062 / #A69D90 / #C44B2E)
- No results: "No institutions found for '{query}'" — 14px, #7A7062

**No animation on hero** — server-rendered page, keep load instant. The `admin-fade-up` stagger belongs to admin-content only.

### Section 2: Value Prop Cards

**Layout:** 3-column grid on desktop (`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-5`), stacked on mobile.

**Cards (3 total):**
1. "Find Your Fees" — links to `/institutions`
2. "Compare Banks" — links to `/fees`
3. "Learn About Fees" — links to `/guides`

**Card anatomy:**
- Container: `rounded-xl border border-[#E8DFD1] bg-white p-6 hover:border-[#C44B2E]/30 hover:shadow-md transition-all duration-300`
- Icon: Lucide icon (16px, #C44B2E), chosen per card: Search, BarChart2, BookOpen
- Card label: 12px, weight 400, uppercase, tracking-[0.1em], #A09788
- Card title: 14px, weight 400, uppercase, tracking-[0.05em], #1A1815
- Card body: 14px, #6B6355, leading-relaxed, 2 sentences max
- Arrow indicator: Lucide `ArrowRight` (14px, #A09788), appears on hover via `group-hover:text-[#C44B2E]`
- Entire card is a `<Link>` wrapper with `group` class for hover coordination

### Section 3: Trust Stats

**Layout:** Horizontal strip — `border-t border-[#E8DFD1] bg-[#F5EFE6]/60`, inner `mx-auto max-w-6xl px-6 py-10`.

**Display as:** 4-item horizontal stat row on desktop, 2x2 grid on mobile.

**Stats (from `getPublicStats()` and `getDataFreshness()`):**
1. `{stats.total_institutions.toLocaleString()}+` — "Banks & Credit Unions"
2. `{stats.total_observations.toLocaleString()}+` — "Fee Observations"
3. "Updated {lastUpdated}" (from `getDataFreshness().last_crawl_at`) — "Data Freshness"
4. "FDIC & NCUA" — "Verified Sources"

**Stat typography:**
- Stat number: 28px, weight 700, #1A1815, tabular-nums
- Stat label: 12px, weight 400, #7A7062, uppercase, tracking-wide

**No hardcoded numbers.** All figures rendered from live DB queries (D-09 from CONTEXT.md).

## Interaction Contracts

### Search Bar
- Debounce: 250ms (existing InstitutionSearchBar behavior — preserve as-is)
- Min query length: 2 characters before API call
- Dropdown: shows up to 10 results (API limit from `/api/institutions`)
- Select: `router.push('/institution/{id}')` — navigates to institution detail page
- Click outside: closes dropdown (existing click-outside handler)
- Keyboard: Tab navigates into dropdown; Enter selects focused result
- No submit button — search is selection-driven, not form-submit-driven

### Value Prop Cards
- Hover: `border-[#C44B2E]/30` border + `shadow-md` + arrow color shift to terracotta
- Transition: `transition-all duration-300`
- Entire card surface is clickable (Link wrapper)

### Trust Stats
- Static display only — no hover, no click, no interaction
- Numbers rendered server-side from DB — no client-side loading state needed

### Page-Level
- No client-side state on the page shell (Server Component)
- Only `InstitutionSearchBar` introduces a client boundary ("use client")
- No animation on page load (server render = instant, no FOUC risk)
- `generateMetadata()` server function provides OpenGraph + canonical tags

## Accessibility

- `<h1>` is unique on the page — only the hero headline
- Value prop cards: each `<Link>` has meaningful text content (no "click here")
- Search input: `<label>` wrapping or `aria-label="Search for a bank or credit union"` on the `<input>`
- Search results dropdown: `role="listbox"`, each result button has `aria-label` with institution name
- Trust stats: `<dl>` / `<dt>` / `<dd>` semantic structure for stat label/value pairs
- Color contrast: #1A1815 on #FAF7F2 = 17.8:1 (AAA). #6B6355 on #FAF7F2 = 5.9:1 (AA). #C44B2E on white = 4.6:1 (AA for large text — only used on CTA button at 14px uppercase = passes AA large)
- Touch targets: all interactive elements minimum 44px height
