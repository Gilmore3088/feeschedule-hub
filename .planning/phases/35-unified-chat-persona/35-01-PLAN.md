<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/research/hamilton | Unauthenticated requests treated as consumer role — no auth required for public tier |
| session → role resolution | Role derived from signed session cookie; forgery not possible without BFI_COOKIE_SECRET |
| admin tools → pipeline ops | triggerPipelineJob only reachable if role resolves to "admin" — premium users get proTools (no ops) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Elevation of Privilege | /api/research/hamilton role resolution | mitigate | Role derived exclusively from getCurrentUser() session result; unauthenticated → consumer always; premium → pro; analyst/admin → admin |
| T-35-02 | Tampering | getHamilton tool gating | mitigate | adminTools includes triggerPipelineJob; gate enforced at AgentConfig construction — consumer/pro configs literally cannot reference those tools |
| T-35-03 | Denial of Service | Cost circuit breaker | mitigate | getDailyCostCents() checked before every request; 503 returned if >= $50/day |
| T-35-04 | Information Disclosure | Consumer role prompt | accept | Consumer prompt is intentionally minimal/public; no sensitive operational data in consumer AgentConfig |
</threat_model>
