- Apply `LIMIT` (default 100, cap at 500)

Create `src/lib/report-assemblers/peer-competitive.ts`:

**Exports:**
```typescript
export interface PeerCompetitiveFilters {
  charter_type?: string;
  asset_tiers?: string[];
  fed_districts?: number[];
}

export interface PeerBriefSection {
  section_type: SectionType;
  title: string;           // conclusion-first title for Hamilton
  data: Record<string, unknown>; // source data Hamilton may reference
  include: boolean;        // false = skip (e.g., fee_change_context with no data)
}

export interface PeerCompetitivePayload {
  data: PeerCompetitiveData;  // feeds renderPeerCompetitiveReport()
  sections: PeerBriefSection[]; // 3-6 sections for generateSection() calls
  manifest: DataManifest;
}

export async function assemblePeerCompetitivePayload(
  filters: PeerCompetitiveFilters
): Promise<PeerCompetitivePayload>
```

**Implementation logic:**

1. Run queries in parallel with `Promise.all`:
   - `getPeerIndex(filters)` — peer segment data
   - `getNationalIndex()` — national benchmarks
   - `getFeeChangeEvents({ charter_type: filters.charter_type, asset_tiers: filters.asset_tiers, fed_districts: filters.fed_districts, limit: 100 })`

2. Build `DataManifest` tracking all three queries (sql string, row_count, executed_at)

3. Build `categories` array by joining peerIndex + nationalIndex by fee_category:
   - Only include categories present in peerIndex (peer segment must have data)
   - Compute `delta_pct = ((peer_median - national_median) / Math.abs(national_median)) * 100` — null if either is null
   - `is_featured = isFeaturedFee(fee_category)`
   - Sort: featured first, then by peer_count descending

4. Derive segment label for title/subtitle (same logic as pro/brief/route.ts — import TIER_LABELS, DISTRICT_NAMES)

5. Build `PeerCompetitiveData` conforming exactly to hamilton/types.ts interface

6. Define `sections: PeerBriefSection[]` — these are inputs for `generateSection()` calls in the Modal worker:
   - Section 1: `{ section_type: 'executive_summary', title: 'Competitive Fee Position — [segment] vs. National Peers', data: { peer_count, top_categories_above_national[], top_categories_below_national[], total_observations }, include: true }`
   - Section 2: `{ section_type: 'peer_competitive', title: 'Featured Fee Analysis — Where [segment] Diverges from Peers', data: { featured_categories (top 6 by |delta_pct|, with peer_median, national_median, delta_pct, peer_count) }, include: true }`
   - Section 3: `{ section_type: 'findings', title: 'Fee Change History — Who Moved First in [segment]', data: { events: feeChangeEvents.slice(0, 20) }, include: feeChangeEvents.length > 0 }` (per D-04 graceful degradation)
   - Section 4: `{ section_type: 'peer_comparison', title: 'Cost Advantage Analysis — Categories Where [segment] Outperforms', data: { below_national: categories.filter(c => c.delta_pct < -5), above_national: categories.filter(c => c.delta_pct > 5) }, include: true }`
   - Section 5: `{ section_type: 'recommendation', title: 'Strategic Positioning — Actions for [segment] Institutions', data: { outlier_categories (delta > 15%), peer_count, segment_label }, include: true }`

7. Compute `data_hash` as SHA-256 of `JSON.stringify({ data, sections_count: sections.length })`

Return `{ data, sections, manifest }`.

Do NOT call `generateSection()` here — the Modal worker calls it. This assembler is pure data.
