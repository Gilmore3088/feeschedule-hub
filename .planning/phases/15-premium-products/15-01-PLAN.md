## Trust Boundaries

| Boundary | Description |

| Browser → /api/reports/generate | Existing auth in place (Phase 13); peer_brief params must be sanitized |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Spoofing | /api/reports/peer-brief/confirm GET | mitigate | getCurrentUser() check before any DB call; 401 if not authenticated |
| T-15-02 | Elevation of Privilege | /api/reports/peer-brief/confirm | mitigate | canAccessPremium() check; 403 if viewer/free role |
| T-15-03 | Tampering | Peer filter params in confirm route | mitigate | fed_districts filtered to 1-12 range; charter validated against allowed values ("bank", "credit_union", ""); asset_tiers passed directly to getPeerIndex() which uses parameterized SQL |
| T-15-04 | Information Disclosure | BriefStatusPoller fetching arbitrary job IDs | accept | Ownership guard on /api/reports/[id]/status returns 403 for non-owners (Phase 13 T-13-11) — client cannot access another user's job |
| T-15-05 | Denial of Service | Debounced confirm fetch (400ms) | accept | Confirm endpoint is read-only and lightweight; existing rate limiting on API routes sufficient |
</threat_model>
