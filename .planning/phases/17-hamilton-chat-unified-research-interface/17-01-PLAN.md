<objective>
Build the Hamilton chat API route and its supporting modules: the agent tool registry (wiring existing DB queries as callable tools) and the Supabase-backed persistent memory layer.

It establishes the core intelligence layer Hamilton uses to answer questions: it can query fee data, pull Fed district data, trigger report generation, and look up institutions. - `src/lib/hamilton/hamilton-agent.ts` — tool registry + system prompt builder
- `src/lib/hamilton/chat-memory.ts` — conversation persistence in Supabase
</objective>

    Create `src/lib/hamilton/chat-memory.ts`:

    Tables to create (via `ensureHamiltonTables()` — call idempotently from the API route on cold start):

    ```sql
    CREATE TABLE IF NOT EXISTS hamilton_conversations (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      user_id INTEGER NOT NULL,
      title TEXT,
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    );

    CREATE TABLE IF NOT EXISTS hamilton_messages (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      conversation_id UUID NOT NULL REFERENCES hamilton_conversations(id) ON DELETE CASCADE,
      role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
      content TEXT NOT NULL,
      token_count INTEGER,
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    );

    CREATE INDEX IF NOT EXISTS idx_hamilton_conv_user ON hamilton_conversations(user_id, updated_at DESC);
    CREATE INDEX IF NOT EXISTS idx_hamilton_msg_conv ON hamilton_messages(conversation_id, created_at ASC);
    ```

    Export the following functions — all use the `sql` client from `@/lib/crawler-db/connection`:

    1. `ensureHamiltonTables(): Promise<void>` — run all CREATE TABLE IF NOT EXISTS statements above. Wrap in try/catch and log errors without throwing (non-blocking for cold start).

    2. `createConversation(userId: number): Promise<string>` — INSERT returning id (UUID string)

    3. `appendMessage(conversationId: string, role: "user" | "assistant", content: string, tokenCount?: number): Promise<void>` — INSERT into hamilton_messages; also UPDATE hamilton_conversations SET updated_at = NOW() for that id

    4. `loadConversationHistory(conversationId: string, limit = 20): Promise<Array<{role: "user"|"assistant", content: string}>>` — SELECT last `limit` messages ORDER BY created_at ASC. Returns plain objects (not UIMessage — the route converts them).

    5. `listConversations(userId: number, limit = 30): Promise<Array<{id: string, title: string|null, updatedAt: string}>>` — SELECT from hamilton_conversations WHERE user_id = userId ORDER BY updated_at DESC LIMIT limit

    6. `updateConversationTitle(conversationId: string, title: string): Promise<void>` — UPDATE hamilton_conversations SET title = $1, updated_at = NOW() WHERE id = $2

    Export interface `ConversationSummary = { id: string; title: string | null; updatedAt: string }`.

Call `ensureHamiltonTables()` at the top of the POST handler (fire-and-forget with `.catch(() => {})`).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/hamilton/chat | User-supplied message content crosses into LLM system prompt context |
| Hamilton tool call → /api/reports/generate | LLM-generated report_type + params passed to internal fetch |
| LLM output → Supabase | Assistant content written to hamilton_messages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | triggerReport tool | mitigate | validate report_type against explicit Set allowlist before internal fetch; reject unknown types with 400 |
| T-17-02 | Elevation of Privilege | /api/hamilton/chat | mitigate | requireAuth("analyst") gate — admin-only like fee-analyst agent; not public |
| T-17-03 | Denial of Service | streamText tool calls | mitigate | stopWhen: stepCountIs(10) caps tool call loops; maxDuration=60 kills hung streams |
| T-17-04 | Information Disclosure | loadConversationHistory | mitigate | query filters by conversation_id + user_id to prevent cross-user history access |
| T-17-05 | Tampering | conversation_id param | mitigate | validate UUID format (regex) before passing to memory layer; reject malformed IDs with 400 |
</threat_model>
