**Gathered:** 2026-04-08

<domain>
## Phase Boundary

Transform the institution slug page from a static fee table into a consumer decision page. Add summary card with rating, interpretation block, visual comparison bars, fee count context, strengths/watch section, enhanced table indicators, comparison hooks, and clean pro CTAs. The page must answer "Is this institution expensive or not?" within 5 seconds.

</domain>

<decisions>
## Implementation Decisions

### Rating Logic (Spec #1)
- **D-01:** Overdraft fee thresholds: ≤$20 = "Consumer-Friendly" (green), $20.01-$36 = "Average Fee Structure" (yellow), >$36 = "Above-Average Fees" (red).
- **D-02:** Bonus friendly signals: fee caps, de minimis policies, balance de minimis, transaction de minimis. These can upgrade the rating or add positive bullets.
- **D-03:** Revenue correlation with rating is a future enhancement -- not in this phase.
- **D-04:** Claude's discretion on the composite formula details, but the overdraft thresholds above are locked.

### Data Dependency Handling (All sections)
- **D-05:** Summary card ALWAYS appears, even when fee categorization is missing. Use fee count + overdraft detection by fee name (not category) as fallback.
- **D-06:** Comparison bars: hidden if no category matches exist. No "data unavailable" messages.
- **D-07:** Table indicators: show arrows (↑↓=) when category match exists, show nothing (not "-") when no match. Hide the column entirely if zero rows have matches.
- **D-08:** Interpretation block: always renders. Can use fee count and overdraft amount even without categories.
- **D-09:** Strengths/Watch: derive from whatever data is available. If only fee count + overdraft, use those.

### Compare Hooks (Spec #8)
- **D-10:** Compare links go to filtered search with pre-applied parameters. E.g., "Compare to nearby credit unions" → `/institutions?state=FL&charter=credit_union`.
- **D-11:** Use existing institution search page -- no new comparison tool needed.

### Layout & Design (Spec #10)
- **D-12:** Center: institution name, summary card, hero elements. Left: table, charts, text blocks.
- **D-13:** 24-32px between sections, clear visual separation between cards.
- **D-14:** Visual hierarchy: Summary (most prominent) → Visual comparisons → Table → CTA.

### CTAs (Spec #7, #9)
- **D-15:** Mid-page CTA after visual comparison section: "Want a deeper breakdown of how this institution generates fee revenue?" → "Unlock full analysis" button.
- **D-16:** Footer CTA: "Financial professionals use Bank Fee Index to benchmark pricing, analyze revenue, and identify opportunities."
- **D-17:** No inline UpgradeGates anywhere. Pro-only sections simply don't render for consumers.

### All Spec Items Are Locked
The full V2 spec at `.planning/phases/30-institution-educational-pages/institution-v2-spec.md` defines all 12 sections. Everything in the spec is a locked decision unless contradicted by the decisions above.

### Claude's Discretion
- Composite rating formula details (beyond the overdraft thresholds)
- Exact wording of interpretation block (must be 2-3 sentences, plain English, no jargon)
- Whether to detect overdraft by fee_name substring matching or a hardcoded name list
- Visual comparison bar normalization approach
- How to compute "typical credit union: 25-35" fee count ranges (hardcoded or computed from data)

</decisions>

<specifics>
## Specific Ideas

- The page must shift from "Here are the fees" to "Here's what this means for you"
- Success: user answers "Is this institution expensive or not?" in 5 seconds
- Summary card is the MOST prominent element on the page
- Rating colors: green = consumer-friendly, yellow = average, red = above-average
- Overdraft thresholds: ≤$20 green, $20.01-$36 yellow, >$36 red
- All copy must match the humanized voice from Phase 29 (no AI-sounding language)

</specifics>
