**WaveReport dataclass:**
```python
@dataclass
class StateResult:
    state_code: str
    before_pct: float      # coverage % before this wave started
    after_pct: float       # coverage % now
    delta_pct: float       # after - before
    fees_added: int        # new extracted_fees rows added during wave
    status: str            # 'complete' | 'failed' | 'skipped'

@dataclass
class WaveReport:
    wave_run_id: int
    campaign_id: str | None
    started_at: str | None
    completed_at: str | None
    states: list[StateResult]
    national_before_pct: float
    national_after_pct: float
    national_delta_pct: float
    total_fees_added: int
    top_url_patterns: list[str]   # top 5 new fee_schedule_url patterns from wave
    top_discoveries: list[str]    # top 5 institution names with new fees this wave
```

**Query functions (all parameterized — no f-string SQL values):**

`_get_coverage_before(conn, wave_run_id, state_code) -> float`
  - Coverage is defined as: COUNT(DISTINCT crawl_target_id) in extracted_fees WHERE crawl_target_id IN (SELECT id FROM crawl_targets WHERE state_code = %s) at the time the wave_state_runs row was created (started_at). Since we can't time-travel the DB, approximate "before" as: count of crawl_targets with fee_schedule_url IS NOT NULL at the time the wave_state_run row was inserted minus the fees_added in agent_runs that ran during this wave. Use a simpler approach: query crawl_targets for this state_code, compute coverage now, subtract fees_added_this_wave from numerator. **Concrete SQL for "before":**
  ```sql
  SELECT
    COUNT(*) FILTER (WHERE fee_schedule_url IS NOT NULL) * 100.0
    / NULLIF(COUNT(*), 0)
  FROM crawl_targets
  WHERE state_code = %s AND status = 'active'
  ```
  Capture this for after. For before: subtract the distinct institutions that got their first fee in this wave from the numerator (query: institutions with fees that only got fees during this wave period using wave_state_runs.started_at).

  **Simplified approach (use this — simpler, good enough):**
  - "after" = current `_get_coverage_pct(conn, state_code)` (copy the query from orchestrator.py)
  - "before" = after - (new_fee_institutions_this_wave / total_active) * 100
  - new_fee_institutions_this_wave = distinct crawl_target_id in extracted_fees WHERE created_at >= wave_state_run.started_at AND crawl_target_id IN (SELECT id FROM crawl_targets WHERE state_code = %s)
  - But only count institutions where that wave was their FIRST fee: WHERE NOT EXISTS (SELECT 1 FROM extracted_fees ef2 WHERE ef2.crawl_target_id = ef.crawl_target_id AND ef2.created_at < wsr.started_at)

`_get_fees_added(conn, wave_run_id, state_code) -> int`
  - JOIN wave_state_runs wsr ON wsr.wave_run_id = %s AND wsr.state_code = %s
  - COUNT(*) FROM extracted_fees ef WHERE ef.crawl_target_id IN (SELECT id FROM crawl_targets WHERE state_code = %s) AND ef.created_at >= wsr.started_at
  - Handle NULL started_at: return 0

`_get_top_url_patterns(conn, wave_run_id, limit=5) -> list[str]`
  - Find fee_schedule_url values for institutions covered in this wave that received new fees during the wave period. Extract domain+path pattern (just use the raw URL, truncated to 80 chars). ORDER BY COUNT(*) DESC, LIMIT limit.
  ```sql
  SELECT ct.fee_schedule_url, COUNT(*) as inst_count
  FROM crawl_targets ct
  JOIN wave_state_runs wsr ON wsr.wave_run_id = %s AND wsr.state_code = ct.state_code
  JOIN extracted_fees ef ON ef.crawl_target_id = ct.id AND ef.created_at >= wsr.started_at
  WHERE ct.fee_schedule_url IS NOT NULL
  GROUP BY ct.fee_schedule_url
  ORDER BY inst_count DESC
  LIMIT %s
  ```
  Return list of `"{url} ({inst_count} insts)"` strings.

`_get_top_discoveries(conn, wave_run_id, limit=5) -> list[str]`
  - Institutions that got new fees this wave, ordered by fees_count DESC (most fees found first)
  ```sql
  SELECT ct.institution_name, ct.state_code, COUNT(ef.id) as fee_count
  FROM crawl_targets ct
  JOIN wave_state_runs wsr ON wsr.wave_run_id = %s AND wsr.state_code = ct.state_code
  JOIN extracted_fees ef ON ef.crawl_target_id = ct.id AND ef.created_at >= wsr.started_at
  GROUP BY ct.id, ct.institution_name, ct.state_code
  ORDER BY fee_count DESC
  LIMIT %s
  ```
  Return list of `"{name} ({state}, {fee_count} fees)"` strings.

`_get_national_coverage(conn) -> float`
  - Single query: COUNT(*) FILTER (WHERE fee_schedule_url IS NOT NULL) * 100.0 / NULLIF(COUNT(*), 0) FROM crawl_targets WHERE status = 'active'

`build_wave_report(conn, wave_run_id: int) -> WaveReport`
  - Call get_wave_run() to get WaveRun (raise ValueError if not found)
  - Query wave_state_runs for all states in this wave
  - For each state, compute StateResult (before/after/delta/fees_added/status)
  - Compute national_before/after/delta (after = current national, before = after - sum of first-fee-institutions/total_active * 100)
  - Collect top_url_patterns and top_discoveries

**Markdown renderer:**

`render_wave_report(report: WaveReport) -> str`
  - Returns a Markdown string. Target: scannable in 60 seconds.
  - Format:
    ```
    # Wave #{wave_run_id} Report
    Campaign: {campaign_id or "standalone"}
    Started: {started_at}  Completed: {completed_at}

    ## National Coverage
    Before: {national_before:.1f}%  After: {national_after:.1f}%  Delta: +{national_delta:.1f}%
    Total fees added: {total_fees_added:,}

    ## Per-State Results
    | State | Before | After | Delta | Fees Added | Status |
    |-------|--------|-------|-------|------------|--------|
    | AL    | 12.3%  | 18.7% | +6.4% | 142        | complete |

    ## Top Discoveries
    1. First National Bank of Wyoming (WY, 23 fees)
    ...

    ## Top URL Patterns
    1. https://example.com/personal/fees (3 insts)
    ...
    ```
  - Sort state rows: complete states first by delta_pct DESC, then failed states.
  - Delta formatting: prefix `+` if positive, `-` if negative, `=` if zero (to 1 decimal place).
  - If no top_discoveries or top_url_patterns, print "None found this wave."

`print_wave_report(conn, wave_run_id: int, output_path: str | None = None) -> None`
  - Calls build_wave_report + render_wave_report
  - Prints rendered report to stdout
  - If output_path is not None, writes to that file (UTF-8)
  - Logs any errors at WARNING level and continues (never raises — called from orchestrator)
  - Wrap in try/except Exception: log.warning("Wave report failed: %s", e) — keeps orchestrator clean

**Coding rules:**
- All SQL parameterized (no f-string values)
- No `console.log` — use `logging.getLogger(__name__)`
- Module-level logger: `log = logging.getLogger(__name__)`
- File stays under 200 lines — keep queries focused
- Guard against NULL started_at in wave_state_runs (return 0 for fees_added)
