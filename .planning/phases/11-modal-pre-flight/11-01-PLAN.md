Implementation outline:

1. **Image** — reuse `pdf_image` definition verbatim (tesseract, poppler, pip from
   requirements.txt, fastapi[standard], add_local_dir). No Playwright needed for preflight.

2. **App attachment** — `app = modal.App("bank-fee-index-workers", image=preflight_image)`.
   Use the same app name as modal_app.py so they coexist under one deployment.

3. **`@app.function` decorator** — `secrets=[modal.Secret.from_name("bfi-secrets")]`,
   `timeout=600` (10 min max for 3-institution run), `image=preflight_image`.
   No schedule — this is on-demand only.

4. **Also expose as HTTP endpoint** — decorate with `@modal.fastapi_endpoint(method="POST")`
   so it can be triggered via curl. Accept a body: `{"state_code": "VT"}` using a Pydantic
   model. Default state_code to `"VT"` (small state, fast seed).

5. **Inside the function body** — inline the full 5-stage pipeline, exactly mirroring
   `pipeline_db` fixture logic in `test_full_pipeline.py`:

   ```
   PREFLIGHT_DB_PATH = "/tmp/preflight_test.db"
   PREFLIGHT_DOC_DIR = "/tmp/preflight_docs"
   SEED_LIMIT = 10
   TARGET_COUNT = 3   # smaller than test suite's 5
   ```

   Config construction (D-03, D-04 — isolation):
   ```python
   config = Config(
       database=DatabaseConfig(path=PREFLIGHT_DB_PATH),
       crawl=CrawlConfig(delay_seconds=0.5, max_retries=2),
       extraction=ExtractionConfig(
           document_storage_dir=PREFLIGHT_DOC_DIR,
           daily_budget_usd=1.0,   # tight budget for pre-flight
       ),
   )
   ```

   NEVER read `os.environ["DATABASE_URL"]` inside preflight_e2e. Assert it is not
   used by adding a comment at the top of the function:
   `# ISOLATION: this function deliberately ignores DATABASE_URL`

   Stage sequence:
   - Stage 1 (seed): `seed_fdic(db, config, limit=SEED_LIMIT)` — filter to state_code
   - Stage 2 (discover): `_discover_one(dict(target), config, False, False)` per target
   - Stage 3 (extract): `_crawl_one(target_dict, config, run_id)` per discovered target
   - Stage 4 (categorize): `categorize_fees.run(db)`
   - Stage 5 (validate): `backfill_validation.run(db, config)`

   Per D-04: wrap everything in a single `with Database(config) as db:` block —
   the Database class opens `/tmp/preflight_test.db` (SQLite) automatically when
   DATABASE_URL is absent from the Config (it is — Config does not read env vars).
   Wait — check `db.py`: `_DATABASE_URL = os.environ.get("DATABASE_URL")` is
   a module-level read. If DATABASE_URL is set in Modal secrets, this will shadow
   the Config path. **Mitigation**: temporarily unset DATABASE_URL for the preflight
   duration using:
   ```python
   import os
   _saved = os.environ.pop("DATABASE_URL", None)
   try:
       with Database(config) as db:
           # ... all stages ...
   finally:
       if _saved is not None:
           os.environ["DATABASE_URL"] = _saved
   ```
   This ensures the Database class uses SQLite via Config even when DATABASE_URL
   is present in the bfi-secrets bundle (D-04).

   Error handling: each stage wrapped in try/except. Append to `errors: list[str]`
   as `f"{stage}:{inst_name}:{str(exc)[:100]}"`. Never raise — always return the
   result dict.

   Return dict (D-07):
   ```python
   return {
       "status": "pass" if fees_extracted >= 1 else "fail",
       "institutions": institutions_seeded,
       "fees_extracted": fees_extracted,
       "duration_s": round(time.time() - t_start, 2),
       "errors": errors,
   }
   ```

   Pass threshold: `fees_extracted >= 1` (at least one fee extracted end-to-end).

6. **Caller convenience**: Add a `__main__` block at the bottom so

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Modal secrets → preflight function | bfi-secrets bundle injects DATABASE_URL + ANTHROPIC_API_KEY into the Modal environment; preflight must not use DATABASE_URL for writes |
| FDIC API → seed stage | External network call; unauthenticated GET; read-only risk |
| Claude Haiku API → extraction stage | ANTHROPIC_API_KEY used; metered by daily_budget_usd cap |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | preflight_e2e / DATABASE_URL | mitigate | Pop DATABASE_URL from os.environ before opening Database, restore in finally block — prevents any code path from writing to production Supabase during preflight |
| T-11-02 | Information Disclosure | /tmp/preflight_test.db | accept | Modal's ephemeral /tmp is container-scoped and destroyed after function exit; no cross-invocation leakage |
| T-11-03 | Denial of Service | extraction stage / daily_budget_usd | mitigate | Cap daily_budget_usd=1.0 in preflight Config; Claude API rejects over-budget calls |
| T-11-04 | Elevation of Privilege | Modal function / secrets | accept | bfi-secrets read-only from Modal's secret store; no write path to secrets from function code |
| T-11-05 | Spoofing | FDIC API response | accept | FDIC API is read-only public data; worst case is stale institution data, not a security issue |
</threat_model>
