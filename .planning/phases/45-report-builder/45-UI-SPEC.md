# Phase 45 — UI Design Contract

> Visual and interaction contract for the Report Builder screen at /pro/reports.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Design target: McKinsey-grade report communication — authoritative, read-only, export-first.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind + CSS vars) |
| Preset | hamilton-editorial (warm parchment tokens) |
| Component library | Radix UI primitives (select, dialog) |
| Icon library | Lucide React |
| Font | Newsreader (serif) for headings, Geist Sans for body |

All styling lives inside `.hamilton-shell` scope. No Tailwind classes that conflict
with `--hamilton-*` CSS custom properties.

## Layout Contract

### Overall Page Structure

Two-panel layout within `.hamilton-shell` main content area (`flex-1 min-w-0 p-6`):

```
┌─────────────────────────────────────────────────────────────┐
│ Left Rail (existing HamiltonLeftRail — unchanged)           │
├─────────────────┬───────────────────────────────────────────┤
│ Config Sidebar  │  Main Content Panel                       │
│ 280px fixed     │  flex-1                                   │
│                 │  Template Gallery (pre-generation)        │
│ Template type   │    OR                                     │
│ Peer set        │  Report Output (post-generation)          │
│ Date range      │                                           │
│ Scenario link   │                                           │
│                 │                                           │
│ [Generate]      │                                           │
│ [Export PDF]    │                                           │
└─────────────────┴───────────────────────────────────────────┘
```

- Config sidebar: `w-[280px] flex-shrink-0` with `border-r` using `--hamilton-border`
- Main panel: `flex-1 min-w-0 overflow-auto`
- Mobile: sidebar collapses above main (stacked), generate button always visible

### Template Gallery State (pre-generation)

4 template cards in a 2×2 grid (`grid grid-cols-2 gap-4`):
- Quarterly Strategy Report
- Peer Brief
- Monthly Pulse
- State Index

Each card: `.hamilton-card` base, `cursor-pointer`, `ring-2 ring-offset-2` when selected

### Report Output State (post-generation)

Single scrollable column reading from top:
1. Report header: title, institution, date range, report type badge
2. Executive Summary section
3. Current vs Proposed Snapshot (if scenario-linked)
4. Strategic Rationale
5. Tradeoff Summary
6. Recommended Position
7. Implementation Notes

All sections separated by `border-b` using `--hamilton-border` + `py-8` spacing

## Component Contracts

### TemplateCard

```tsx
interface TemplateCardProps {
  type: 'quarterly_strategy' | 'peer_brief' | 'monthly_pulse' | 'state_index';
  title: string;
  description: string;           // 1–2 sentences
  estimatedTime: string;         // "~30 seconds"
  isSelected: boolean;
  onClick: () => void;
}
```

- Base: `.hamilton-card p-4 cursor-pointer transition-all`
- Selected: `ring-2 ring-offset-2` with terracotta ring color
- Hover (unselected): `--hamilton-shadow-elevated`
- Title: `text-base font-semibold` Geist Sans
- Description: `text-sm` `--hamilton-text-secondary`
- Type badge: `text-[10px] uppercase tracking-wider` in `--hamilton-accent-subtle` bg with `--hamilton-accent` text

### ConfigSidebar

- Wraps Radix Select for peer set + date range
- Date range: two `<input type="date">` fields, labeled "From" / "To"
- Scenario selector: Radix Select populated from `hamilton_scenarios` — label shows `fee_category + " scenario"`, value is UUID
- "Link a Scenario (optional)" — section with subtle label, not required
- Generate button: full-width, `bg-gradient(--hamilton-gradient-cta)` terracotta, `text-white font-semibold`
- Export PDF button: appears ONLY after report is generated, outlined style with Lucide `Download` icon

### ReportSection

Each section in the read-only report output:
- Section heading: h2, Newsreader serif, `text-xl font-semibold mb-4`
- Narrative: `text-[15px] leading-relaxed text-[--hamilton-text-primary]`
- Stat callout boxes (for snapshot/tradeoffs): `.hamilton-card p-4` with large Newsreader number + small label below

### StatCalloutBox

For "Current vs Proposed Snapshot" and "Tradeoff Summary":
```
┌──────────────────────────┐
│  Monthly Maintenance     │
│  CURRENT     PROPOSED    │
│  $12.00   →  $10.00      │ ← Newsreader 32px, tabular-nums
│  P60 national            │ ← 11px caption
└──────────────────────────┘
```

### GeneratingState

While Hamilton is generating (streaming):
- Show spinner with copy: "Hamilton is writing your report..."
- Progressive section reveal: each section fades in as it completes
- No skeleton shimmer — use a clean loading message at the top of the content area

### EmptyState (no report generated yet)

Shown in the main panel when no template is selected or report not yet generated:
- Icon: Lucide `FileText` at 48px, `--hamilton-text-tertiary`
- Heading: "Select a report template" (Newsreader serif, 20px)
- Body: "Choose a template from the sidebar, configure your scope, and Hamilton will generate a McKinsey-grade executive summary."

## Interaction States

### Template Selection
- Click template card → card gets selected ring, sidebar "Generate Report" button activates
- Only one template selected at a time (radio behavior)
- Re-clicking same template deselects (returns to empty state)

### Report Generation Flow
1. User fills config sidebar (template + optional peer set + date range + optional scenario)
2. Clicks "Generate Report"
3. Button shows loading spinner, text changes to "Generating..."
4. Main panel shows generating state (spinner + copy)
5. Sections stream in progressively as Hamilton writes them
6. On complete: full report visible, "Export PDF" button activates in sidebar
7. On error: error state replaces spinner, button resets to "Generate Report"

### PDF Export
- Click "Export PDF" → triggers server action that renders @react-pdf/renderer PDF
- While rendering: button shows spinner, text: "Preparing PDF..."
- On complete: browser download triggered (filename: `hamilton-report-{date}.pdf`)
- On error: toast notification "PDF generation failed"

### Scenario Linking
- Scenario selector shows only active scenarios for the current user
- When scenario selected: snapshot section auto-populates with `current_value` / `proposed_value` from `hamilton_scenarios` row
- If no scenarios exist: selector shows placeholder "No scenarios saved yet" and is disabled

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none — not using shadcn for new components | not required |
| Radix UI | Select, Dialog (if needed for confirmation) | Wrapped in existing pattern (see AnalyzeWorkspace) |
| @react-pdf/renderer | Document, Page, Text, View, StyleSheet | Server-side only via API route — never in client bundle |
| Lucide React | FileText, Download, Loader2, CheckCircle | Standard project usage |

@react-pdf/renderer MUST be rendered server-side only (API route `/api/pro/report-pdf`).
Client bundle must never import `@react-pdf/renderer` directly — only call the API route.

## Accessibility Contract

- Template cards: `role="radio"` with `aria-checked` state
- Template card group: `role="radiogroup"` with `aria-label="Report template"`
- Generate button: `aria-busy="true"` while generating
- Report sections: landmark `<section>` with `aria-label` matching section heading
- Export button: `aria-label="Export report as PDF"` when icon-only (mobile)
- Focus trap: none required (no modal overlay in base flow)

---

## Read-Only Enforcement

Per D-06 (screen boundary rule):
- Generated report output area: NO `<input>`, NO `<select>`, NO `<textarea>`, NO `<button>` inside
- Export button lives in the CONFIG SIDEBAR, not inside the report output
- "This report is read-only" notice shown as subtle caption below report title
- No inline editing affordances (no pencil icons, no hover edit states)
