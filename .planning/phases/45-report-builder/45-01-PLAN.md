# Plan 45-01: Install @react-pdf/renderer and update next.config.ts

## Goal

Install the `@react-pdf/renderer` package (not currently in package.json) and update `next.config.ts` to mark it as a server-external package so it never enters the client bundle.

## Context

`@react-pdf/renderer` uses Node.js-only APIs and cannot run in the browser. It must be rendered server-side only. The UI-SPEC explicitly requires `serverExternalPackages: ['@react-pdf/renderer']` in next.config.ts (Registry Safety section).

## Tasks

### Task 1: Install @react-pdf/renderer

<read_first>
- `package.json` — verify @react-pdf/renderer is absent, check existing dependency versions
</read_first>

<action>
Run the following command from the project root:

```bash
npm install @react-pdf/renderer
```

This adds `@react-pdf/renderer` to `package.json` dependencies and updates `package-lock.json`.

Expected result: `package.json` dependencies now contains `"@react-pdf/renderer": "^X.X.X"`.
</action>

<acceptance_criteria>
- `package.json` contains `"@react-pdf/renderer"` in the dependencies object
- `node_modules/@react-pdf/renderer` directory exists
- `npm install` exits with code 0
</acceptance_criteria>

---

### Task 2: Update next.config.ts with serverExternalPackages

<read_first>
- `next.config.ts` — read full file to understand current config structure before editing
</read_first>

<action>
Add `serverExternalPackages: ['@react-pdf/renderer']` to the `nextConfig` object in `next.config.ts`.

The config object currently has `output: "standalone"` and `poweredByHeader: false` and `async headers()` and `async rewrites()`. Add the new key directly after `poweredByHeader`:

```typescript
const nextConfig: NextConfig = {
  output: "standalone",
  poweredByHeader: false,
  serverExternalPackages: ['@react-pdf/renderer'],
  // ... rest of config unchanged
};
```
</action>

<acceptance_criteria>
- `next.config.ts` contains the string `serverExternalPackages: ['@react-pdf/renderer']`
- `npx tsc --noEmit` exits with code 0 (no TypeScript errors introduced)
</acceptance_criteria>
