Create a new server component `hero-cards.tsx` (per D-04, D-05, D-06):

**Component props:**
```typescript
interface HeroCardsProps {
  financials: InstitutionFinancial[];  // from getFinancialsByInstitution, newest-first
  peerRanking: PeerRanking | null;     // from getInstitutionPeerRanking
}
```

**Data preparation (CRITICAL -- per Pitfall 1 in RESEARCH.md):**
- The `financials` array arrives newest-first (DESC). Sparklines need oldest-first.
- Create reversed copy: `const sorted = [...financials].reverse()`
- Extract sparkline arrays: `sorted.map(f => f.total_assets).filter((v): v is number => v !== null && v > 0)`
- Do the same for: total_deposits, service_charge_income, total_revenue, efficiency_ratio, fee_income_ratio

**Latest values:**
- `const latest = financials[0]` (newest quarter, since financials is DESC)
- Compute QoQ delta for each metric: `((latest - prev) / prev * 100).toFixed(1)` where prev = financials[1]
- Compute YoY delta: find the quarter 4 positions back in the array (same quarter last year)

**Peer context (per D-06):**
- If peerRanking exists, show percentile: `P{Math.round(((peerRanking.peer_count - peerRanking.sc_rank + 1) / peerRanking.peer_count) * 100)} among {peerRanking.tier} peers`
- Apply peer badge to the SC Income card specifically (that is what peerRanking provides)
- For other metrics, show QoQ/YoY delta as the sub-label instead

**Staleness badge (Claude's Discretion):**
- Compute days since latest report_date. If > 95 days, show amber staleness badge.
- `const daysSince = (Date.now() - new Date(latest.report_date).getTime()) / (1000 * 60 * 60 * 24)`
- Badge text: "Data may be stale" in `bg-amber-50 text-amber-700 rounded px-1.5 py-0.5 text-[10px] font-bold`

**Card layout (per D-05):**
- 6 hero stat cards in a responsive grid: `grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-6 gap-3`
- Each card uses the `.admin-card` class with `p-4`
- Card structure:
  1. Label: `text-[10px] font-semibold text-gray-400 uppercase tracking-wider`
  2. Value row: `flex items-end gap-3 mt-1` containing:
     - Large number: `text-2xl font-bold tabular-nums text-gray-900 dark:text-gray-100`
     - Sparkline: `<Sparkline data={sparklineArray} width={64} height={24} color="#3b82f6" />`
  3. Sub-label: `text-[11px] text-gray-400 mt-1` showing peer percentile or QoQ delta
  4. Delta indicator: emerald for positive (growth), red for negative (decline). Use `text-emerald-600` / `text-red-600`.

**6 cards (per D-04):**
1. **Total Assets** -- `formatAssets(latest.total_assets)` -- sparkline of quarterly assets -- QoQ delta
2. **Total Deposits** -- `formatAssets(latest.total_deposits)` -- sparkline -- QoQ delta
3. **SC Income** -- `formatAmount(latest.service_charge_income)` -- sparkline -- **peer percentile badge** (from peerRanking)
4. **Net Income** -- `formatAmount(latest.total_revenue)` (total_revenue is closest to net income in schema; if null, show "N/A") -- sparkline -- YoY delta
5. **Efficiency Ratio** -- `latest.efficiency_ratio?.toFixed(1)%` or "N/A" -- sparkline -- QoQ delta. Lower is better, so delta colors are inverted (decrease = emerald).
6. **Fee/Deposit Ratio** -- `latest.fee_income_ratio?.toFixed(2)%` or "N/A" -- sparkline -- peer median comparison from peerRanking.peer_median_fee_ratio

**Empty state:**

**Formatting (use existing helpers from format.ts):**
- Assets: `formatAssets()` -- handles billions/millions automatically
- Amounts: `formatAmount()` -- handles thousands with commas
- Note: DB stores monetary values in thousands. `formatAssets` expects values in dollars. Check: if `total_assets` stored as thousands, multiply by 1000 before `formatAssets()`. Verify against existing page behavior.

**Do NOT add "use client"** -- this is a server component. Sparkline is already a client component that can be used from server components.
