  quarter: string;               // e.g. "Q1 2026"
  total_institutions: number;
  total_bank_institutions: number;

**assembleNationalQuarterly() function** — async, no parameters:

1. Record `assembled_at = new Date().toISOString()`.
2. Call `getNationalIndex()` — all non-rejected fees (includes staged + pending per existing behavior). Track as manifest query entry.
3. Call `getPeerIndex({ charter_type: 'bank' })` to get bank-only medians. Track as manifest query entry.
4. Call `getPeerIndex({ charter_type: 'credit_union' })` to get CU-only medians. Track as manifest query entry.
5. Call `getBeigeBookHeadlines()` — returns Map<district, { text, release_date }>. Track as manifest query entry (row count = map size).
6. Build `categories` array by iterating the national index entries. For each entry:
   - Look up bank median from bank-only IndexEntry for same fee_category (use a Map keyed by fee_category)
   - Look up CU median from CU-only IndexEntry
   - `display_name` from `getDisplayName(entry.fee_category)`
   - bank_median/cu_median: use the `median_amount` from the charter-filtered result (null if category not present)
7. Build `district_headlines` by spreading the BeigeBookHeadlines map (districts 1–12). Only include districts that have a headline. Trim headline text to 500 chars for safety.
8. Compute `data_hash` using `createHash('sha256')` from Node `crypto` module over `JSON.stringify({ categories, district_headlines })`. Use `pipeline_commit` from `process.env.VERCEL_GIT_COMMIT_SHA ?? 'local'`.
9. Derive `quarter` from current date: Q1=Jan-Mar, Q2=Apr-Jun, Q3=Jul-Sep, Q4=Oct-Dec.
10. Count `total_institutions` from the Set of unique institution_counts across all categories (use the national index's max institution_count as a proxy — it represents the broadest coverage).

**Manifest query entries format** — each query object:
```typescript
{ sql: 'getNationalIndex()', row_count: entries.length, executed_at: assembled_at }
```
Use the function name as `sql` (not raw SQL — these are TypeScript query wrappers, not raw Postgres statements).
