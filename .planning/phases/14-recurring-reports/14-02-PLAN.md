**assembleStateFeeIndex(stateCode: string) function** — async:

1. Normalize `stateCode = stateCode.toUpperCase()` (defensive).
2. Look up `state_name` from a simple state code → name mapping (inline the 50-state map as a constant — do not import from an external source). If state not found, throw `Error(\`Unknown state code: ${stateCode}\`)`.
3. Look up `fed_district` number for this state. Inspect `src/lib/fed-districts.ts` — use whatever export provides this mapping. If fed-districts.ts does not export a lookup by state_code, build a local STATE_TO_DISTRICT map (12 districts, each covering a group of states — use the canonical Fed district assignments).
4. Record `assembled_at = new Date().toISOString()`.
5. Call `getPeerIndex({ state_code: stateCode })` — state-level fees. Track manifest entry.
6. Call `getNationalIndex()` — national baseline. Track manifest entry.
7. If `fed_district !== null`: call `getBeigeBookHeadline(fed_district)` and `getDistrictIndicators(fed_district)`. Track each as manifest entry.
8. Build `categories` array by merging state + national data:
   - Build nationalMap: `Map<string, IndexEntry>` keyed by fee_category
   - For each state IndexEntry: compute `delta_pct` using the market.ts formula: `((state_median - nat_median) / nat_median) * 100` rounded to 1dp. Null if either median is null or nat_median is zero.
   - Only include categories that exist in the state index (categories with zero state observations are omitted — no point showing a state has no data for a fee)
   - `display_name` from `getDisplayName(entry.fee_category)`
9. Sort `categories` by `Math.abs(delta_pct ?? 0)` descending — most divergent categories first (most informative for executives).
10. Build `district_indicators` from `getDistrictIndicators()` result. Map each indicator to `{ series_id, value, period }`. Cap at 10 indicators maximum.
11. Compute `data_hash` via Node `crypto.createHash('sha256')` over `JSON.stringify({ categories, state_code: stateCode })`. `pipeline_commit` from `process.env.VERCEL_GIT_COMMIT_SHA ?? 'local'`.

Export: `assembleStateFeeIndex`, `StateFeeIndexPayload`, `StateFeeIndexCategory`.

**DeltaPill formatting (D-07)** — the `delta_pct_display` cell value is a pre-formatted string with inline HTML for color. Because `dataTable` renders via `formatCell` which calls `escapeHtml`, use a custom approach: add a helper function `formatDeltaCell(delta: number | null): string` that returns:
- null → "—"
- positive (above national) → `<span style="color:#dc2626">+{val}%</span>` (red — cost disadvantage)
- negative (below national) → `<span style="color:#059669">{val}%</span>` (emerald — cost advantage)
- zero → `<span style="color:#6b7280">0.0%</span>` (gray)

Since `dataTable`'s `formatCell` calls `escapeHtml` on string values, you cannot inject HTML via the `rows` map. Instead, render the delta column manually by building a custom table HTML string for this section. Do NOT use the generic `dataTable()` component for the delta table — write a `renderDeltaTable(categories: StateFeeIndexCategory[]): string` helper function inside the template file that builds the `<table>` HTML directly with proper color styling and escapes category/display names.
