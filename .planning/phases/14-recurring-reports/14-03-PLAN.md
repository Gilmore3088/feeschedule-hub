export interface MonthlyPulsePayload {
  report_date: string;       // ISO date string
  period_label: string;      // e.g. "April 2026"
  movers_up: PulseMover[];   // categories that increased, sorted by |change_pct| desc
  movers_down: PulseMover[]; // categories that decreased, sorted by |change_pct| desc
  total_categories_tracked: number;
  total_movers: number;
  manifest: DataManifest;
}

**assembleMonthlyPulse() function** — async, no parameters:

Movement detection strategy: Since the pipeline does not guarantee a timestamped snapshot from exactly 30 days ago, use the `fee_index_cache` if available (cached via `publish-index` command) as "prior period" vs. live `getNationalIndex()` as "current period". If `fee_index_cache` does not exist or is empty, compute movement against a synthetic prior by using the existing `getNationalIndexCached()` as prior and live `getNationalIndex()` as current.

Concretely:
1. Call `getNationalIndex()` → `currentIndex`. Track manifest entry.
2. Call `getNationalIndexCached()` → `cachedIndex`. Track manifest entry. If cached and live are identical (same data, no pipeline run between them), `movers_up` and `movers_down` will both be empty — this is correct behavior.
3. Build currentMap and cachedMap keyed by fee_category.
4. For each category in currentIndex:
   - Look up prior `median_amount` from cachedMap (null if not present)
   - Compute `change_pct = ((current - prior) / prior) * 100` rounded to 1dp
   - Classify direction: `change_pct > 1.0` → "up", `change_pct < -1.0` → "down", else "flat"
   - Only include in movers if `Math.abs(change_pct) > 5.0` — signal, not noise (per D-08 "what moved")
5. Split into `movers_up` (direction="up") and `movers_down` (direction="down"), sorted by |change_pct| desc within each.
6. `period_label` from current month: `new Date().toLocaleDateString('en-US', { month: 'long', year: 'numeric' })`
7. `total_categories_tracked` = currentIndex.length
8. `total_movers` = movers_up.length + movers_down.length
9. Compute `data_hash` via Node crypto sha256 over `JSON.stringify({ movers_up, movers_down })`. `pipeline_commit` from `process.env.VERCEL_GIT_COMMIT_SHA ?? 'local'`.

Export: `assembleMonthlyPulse`, `MonthlyPulsePayload`, `PulseMover`.

The pulse cron runs on the 1st of each month at 08:00 UTC: `modal.Cron("0 8 1 * *")`. No daily cron conflict — this is a monthly schedule on a different hour.

**Function design:**

The pulse generation cannot call `assembleMonthlyPulse()` directly (that's TypeScript). Instead, the Modal function triggers the Next.js report generation API — the same pattern used for any report type. The API handles assembly, Hamilton calls, and the generate_report worker.

The function:
1. Reads `NEXT_PUBLIC_APP_URL` env var (must be set in bfi-secrets) — this is the base URL for the Next.js app (e.g. `https://bankfeeindex.com`).
2. POSTs to `{NEXT_PUBLIC_APP_URL}/api/reports/generate` with body `{ "report_type": "monthly_pulse" }`.
3. For authentication: the API route requires a session. For cron-triggered generation, use a `REPORT_CRON_SECRET` env var passed as a `X-Cron-Secret` header. The Next.js generate route must accept this header as an alternative auth path (add the header check to the generate route in the same commit — see below).
4. Reads the 202 response `{ jobId }` and returns `{ triggered: True, job_id: jobId, period: datetime.now().strftime("%B %Y") }`.
5. On HTTP error: returns `{ triggered: False, error: response.text, status_code: response.status_code }` (does not raise — cron failures should not crash Modal, just log).

**Cron secret auth update to Next.js generate route** (`src/app/api/reports/generate/route.ts`):
After the existing `getCurrentUser()` call, add a fallback: if `getCurrentUser()` returns null, check for the `X-Cron-Secret` header. If header value matches `process.env.REPORT_CRON_SECRET` (and `REPORT_CRON_SECRET` is non-empty), allow the request as a cron-authenticated call with `user_id: null`. **Python function code to append to modal_app.py:**

```python
@app.function(
    schedule=modal.Cron("0 8 1 * *"),
    timeout=60,
    secrets=secrets,
)
def run_monthly_pulse():
    """Monthly pulse report: triggers generation on the 1st of each month at 08:00 UTC."""
    import os
    import json
    import urllib.request
    import urllib.error
    from datetime import datetime, timezone

    app_url = os.environ.get("NEXT_PUBLIC_APP_URL", "")
    cron_secret = os.environ.get("REPORT_CRON_SECRET", "")
    if not app_url:
        return {"triggered": False, "error": "NEXT_PUBLIC_APP_URL not set"}
    if not cron_secret:
        return {"triggered": False, "error": "REPORT_CRON_SECRET not set"}

    endpoint = f"{app_url.rstrip('/')}/api/reports/generate"
    payload = json.dumps({"report_type": "monthly_pulse"}).encode()

    req = urllib.request.Request(
        endpoint,
        data=payload,
        headers={
            "Content-Type": "application/json",
            "X-Cron-Secret": cron_secret,
        },
        method="POST",
    )

    try:
        with urllib.request.urlopen(req, timeout=30) as resp:
            body = json.loads(resp.read().decode())
            period = datetime.now(timezone.utc).strftime("%B %Y")
            return {"triggered": True, "job_id": body.get("jobId"), "period": period}
    except urllib.error.HTTPError as exc:
        return {"triggered": False, "error": exc.read().decode()[:500], "status_code": exc.code}
    except Exception as exc:
        return {"triggered": False, "error": str(exc)[:500]}
```
