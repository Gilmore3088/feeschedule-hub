    **Step 2: Create handleExternal function**

    Add after the existing handler functions:

    ```typescript
    async function handleExternal(
      query: string | undefined,
      view: string | undefined,
      limit: number
    ) {
      // If query provided, do full-text search
      if (query) {
        const category = view || undefined; // view param doubles as category filter
        const results = await searchExternalIntelligence(query, category ? { category } : undefined);
        return {
          query,
          results: results.map((r) => ({
            source_name: r.source_name,
            source_date: r.source_date,
            category: r.category,
            tags: r.tags,
            headline: r.headline,
            content_snippet: r.content_text.substring(0, 500),
            source_url: r.source_url,
            citation: `[Source: ${r.source_name}, ${r.source_date}]`,
          })),
          total: results.length,
        };
      }

      // No query — list recent entries
      const { items, total } = await listIntelligence(limit, 0);
      return {
        items: items.map((r) => ({
          source_name: r.source_name,
          source_date: r.source_date,
          category: r.category,
          tags: r.tags,
          content_snippet: r.content_text.substring(0, 300),
          source_url: r.source_url,
          citation: `[Source: ${r.source_name}, ${r.source_date}]`,
        })),
        total,
      };
    }
    ```

    Key design choices:
    - Each result includes a pre-formatted `citation` field so Hamilton can paste it directly (per D-02)
    - The `view` parameter is reused as category filter (no new param needed)
    - Content is truncated to 500 chars for search results, 300 for listing, to avoid bloating tool responses
    - Return structure matches the pattern of other handlers (data + metadata)
