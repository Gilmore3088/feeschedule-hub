  <action>
    Create `scripts/migrations/027-external-intelligence.sql` with an idempotent migration (per D-03) that:

    1. CREATE TABLE IF NOT EXISTS external_intelligence:
       - id: SERIAL PRIMARY KEY
       - source_name: TEXT NOT NULL
       - source_date: DATE NOT NULL
       - category: TEXT NOT NULL CHECK (category IN ('research', 'survey', 'regulation', 'news', 'analysis'))
       - tags: TEXT[] NOT NULL DEFAULT '{}'
       - content_text: TEXT NOT NULL
       - source_url: TEXT (nullable)
       - created_at: TIMESTAMPTZ NOT NULL DEFAULT NOW()
       - created_by: TEXT (nullable)
       - search_vector: TSVECTOR

    2. CREATE INDEX (IF NOT EXISTS) on search_vector using GIN for full-text search
    3. CREATE INDEX (IF NOT EXISTS) on category for filtered queries
    4. CREATE INDEX (IF NOT EXISTS) on source_date DESC for chronological listing
    5. CREATE INDEX (IF NOT EXISTS) on tags using GIN for array containment queries

    6. Create a trigger function that auto-updates search_vector on INSERT or UPDATE (per D-05):
       ```sql
       search_vector = setweight(to_tsvector('english', COALESCE(NEW.source_name, '')), 'A') ||
                       setweight(to_tsvector('english', COALESCE(NEW.content_text, '')), 'B')
       ```

    7. CREATE TRIGGER (OR REPLACE if Postgres 14+, otherwise use IF NOT EXISTS pattern with a DO block) to fire BEFORE INSERT OR UPDATE.

    Use DO $$ blocks for idempotency where CREATE IF NOT EXISTS is not available (triggers). Follow the pattern from 025-beige-book-themes.sql.
