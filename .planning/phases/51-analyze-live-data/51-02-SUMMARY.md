## What Was Built

**AnalysisPdfDocument** (`src/components/hamilton/reports/AnalysisPdfDocument.tsx`): A new server-side-only react-pdf Document component that renders an `AnalyzeResponse` as a professional PDF. Uses the exact same COLORS constant from PdfDocument.tsx (`textPrimary: "#1c1917"`, `accent: "#b45309"`, etc.). Layout has 5 sections: Hamilton's View (multi-paragraph, split on double newlines), What This Means, Why It Matters (bullet list with `--` prefix), Evidence (statCalloutGrid boxes), and a fixed footer. No CSS variables used — react-pdf cannot resolve them.

**PDF route dispatch** (`src/app/api/pro/report-pdf/route.ts`): The existing endpoint now reads `body.type` before branching. `type: "analysis"` validates `hamiltonView` presence, renders `AnalysisPdfDocument`, and returns `hamilton-analysis-YYYY-MM-DD.pdf`. Default/`"report"` path is unchanged. Single endpoint serves both content types.

**Export PDF button** (`src/components/hamilton/analyze/AnalyzeCTABar.tsx`): `onExportPdf?: () => void` and `isExporting?: boolean` added to props interface. The button renders in the CTA bar when `onExportPdf` is provided, matching the existing outlined secondary button design (same border/hover/color style). Shows "Exporting..." during request.

**handleExportPdf** (`src/components/hamilton/analyze/AnalyzeWorkspace.tsx`): `useCallback` that POSTs `parsedResponse` to `/api/pro/report-pdf` with `type: "analysis"`, receives the PDF blob, creates an object URL, triggers a browser download anchor click, then cleans up. `isExporting` state guards against double-clicks. The `AnalyzeResponse` shape is assembled from `parsedResponse` fields using `satisfies` for type safety.

## Decisions Made

1. **Single endpoint, type dispatch** — Extended `/api/pro/report-pdf` with a `body.type` branch rather than creating a new route. Keeps PDF logic co-located and reuses auth, error handling, and renderToBuffer boilerplate.

2. **COLORS copied verbatim** — react-pdf's StyleSheet cannot resolve CSS custom properties, so the COLORS object is duplicated from PdfDocument.tsx. Any future brand color change must be applied to both files. Accepted as a known tradeoff per D-12.

3. **onExportPdf opt-in** — The button only renders when the prop is provided, keeping AnalyzeCTABar composable for future screens that may not need PDF export.
