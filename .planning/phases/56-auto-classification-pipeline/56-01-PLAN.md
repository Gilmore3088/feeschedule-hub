| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01 | Tampering | fee_name input to classify_fee() | mitigate | normalize_fee_name() strips special chars/punctuation before any lookup; parameterized queries (%s / ?) prevent SQL injection at INSERT |
| T-56-02 | Tampering | canonical_fee_key output from classify_fee() | accept | classify_fee() only returns keys from CANONICAL_KEY_MAP (fixed dict); no user-supplied keys can enter the column. Low risk — deterministic dict lookup. |
| T-56-03 | Denial of Service | classify_fee() called per-fee in INSERT loop | accept | classify_fee() is pure in-memory dict lookup + regex, O(1) per call. No I/O. Cannot cause timeout or resource exhaustion. |
