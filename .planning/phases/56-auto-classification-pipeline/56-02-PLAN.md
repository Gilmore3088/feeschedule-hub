-- Phase 56: classification_cache for LLM fallback results (D-03)
-- One row per unique normalized fee name. Prevents repeat Haiku API calls.
-- Same fee name string never triggers a second LLM call.

CREATE TABLE IF NOT EXISTS classification_cache (
    normalized_name TEXT PRIMARY KEY,
    canonical_fee_key TEXT,
    confidence FLOAT NOT NULL,
    model TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Index for querying by canonical_fee_key (e.g., "find all names classified as overdraft")
CREATE INDEX IF NOT EXISTS idx_classification_cache_key
    ON classification_cache(canonical_fee_key)
    WHERE canonical_fee_key IS NOT NULL;

-- Index for finding low-confidence entries that may need re-classification
CREATE INDEX IF NOT EXISTS idx_classification_cache_low_conf
    ON classification_cache(confidence)
    WHERE confidence < 0.90;
```

"""Batch LLM classification for fees with canonical_fee_key IS NULL.

Post-crawl batch job (D-01): queries all extracted_fees with NULL canonical_fee_key,
deduplicates by normalized name, checks classification_cache, sends misses to
Claude Haiku in batches of 50, writes cache entries, updates extracted_fees.

Usage:
    python -m fee_crawler classify-nulls          # dry-run
    python -m fee_crawler classify-nulls --fix    # apply classifications
"""

import logging
import os

import anthropic
import psycopg2
import psycopg2.extras

from fee_crawler.fee_analysis import (
    CANONICAL_KEY_MAP,
    NEVER_MERGE_PAIRS,
    normalize_fee_name,
)

log = logging.getLogger(__name__)

BATCH_SIZE = 50  # Fee names per Haiku request (Claude's Discretion)
CONFIDENCE_THRESHOLD = 0.90  # D-02: auto-approve threshold
MODEL = "claude-haiku-4-5-20251001"
```

Key functions to implement:

1. `classify_with_cache(conn, normalized_name: str) -> tuple[str | None, float]`:
   - SELECT from classification_cache WHERE normalized_name = %s
   - Return (canonical_fee_key, confidence) on hit, (None, 0.0) on miss

2. `write_cache_entry(conn, normalized_name: str, canonical_key: str | None, confidence: float, model: str)`:
   - INSERT INTO classification_cache ... ON CONFLICT (normalized_name) DO UPDATE SET ...
   - conn.commit() after write

3. `_validate_llm_result(normalized_name: str, suggested_key: str) -> bool`:
   - Check suggested_key exists in CANONICAL_KEY_MAP. If not, return False.
   - Check NEVER_MERGE_PAIRS: for each pair (a, b), if normalized_name contains one and suggested_key matches the other, return False.
   - Return True if passes both checks.

4. `_classify_batch_with_llm(names: list[str]) -> list[dict]`:
   - Build prompt with CLASSIFY_SYSTEM prompt and list of fee names
   - Call anthropic.Anthropic().messages.create() with model=MODEL, tool_use pattern
   - Tool schema: extract_classifications tool returning list of {fee_name, canonical_fee_key, confidence}
   - Parse response, return list of dicts

   System prompt:
   ```
   You are a bank fee taxonomy specialist. For each fee name, identify the canonical
   fee category from the approved taxonomy. Only use canonical keys from the provided list.
   If a fee does not match any canonical category, respond with null and confidence 0.0.
   ```

   The tool input_schema should accept an array of objects with fee_name, canonical_fee_key (nullable string), and confidence (number).

   Include the list of valid canonical keys (from CANONICAL_KEY_MAP.keys()) in the user message so Haiku knows the valid options.

5. `run(conn, *, fix: bool = True) -> dict`:
   - Query: `SELECT DISTINCT fee_name FROM extracted_fees WHERE canonical_fee_key IS NULL AND review_status != 'rejected'`
   - For each unique fee_name, call normalize_fee_name() to get the cache key
   - Check classification_cache for each normalized name
   - Collect cache misses into batches of BATCH_SIZE
   - For each batch, call _classify_batch_with_llm()
   - For each LLM result: validate with _validate_llm_result(), write_cache_entry()
   - If confidence >= CONFIDENCE_THRESHOLD and validation passes and fix=True:
     UPDATE extracted_fees SET canonical_fee_key = %s, variant_type = NULL WHERE fee_name = %s AND canonical_fee_key IS NULL
   - Return dict: {total_null: int, cache_hits: int, llm_classified: int, below_threshold: int, never_merge_rejected: int, invalid_key_rejected: int}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Haiku LLM output -> classify_nulls | LLM may return hallucinated canonical_fee_key values or incorrect confidence scores |
| Fee names -> Haiku prompt | Fee names extracted from bank documents enter the LLM prompt as untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-04 | Tampering | LLM returns invalid canonical_fee_key | mitigate | _validate_llm_result() checks every LLM-returned key against CANONICAL_KEY_MAP dict; unknown keys rejected regardless of confidence |
| T-56-05 | Tampering | LLM conflates NSF/overdraft or other NEVER_MERGE pairs | mitigate | _validate_llm_result() checks NEVER_MERGE_PAIRS; if fee name contains one member and LLM suggests the other, result rejected and fee goes to human review |
| T-56-06 | Information Disclosure | ANTHROPIC_API_KEY in environment | accept | Key injected via Modal secrets at runtime; never logged or stored in code; anthropic SDK reads from env var |
| T-56-07 | Spoofing | Prompt injection via fee name in Haiku batch | mitigate | normalize_fee_name() strips special characters and punctuation before name enters the prompt; batch prompt uses XML delimiters around fee names |
| T-56-08 | Tampering | LLM returns inflated confidence score | mitigate | Confidence score is validated (0.0-1.0 range); but fundamentally the LLM controls the confidence value. Mitigation: human review queue catches sub-0.90 results. Accept residual risk for >= 0.90 results since CANONICAL_KEY_MAP validation catches wrong keys. |
</threat_model>
