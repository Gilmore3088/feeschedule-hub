    Create the `queryNationalData` tool per D-01 (unified), D-02 (source categories), D-03 (facade over existing functions).

    **Zod input schema:**
    ```typescript
    z.object({
      source: z.enum(["call_reports", "economic", "health", "complaints", "fee_index", "derived"])
        .describe("Data source category to query"),
      view: z.string().optional()
        .describe("Specific view within the source (e.g., 'trend', 'by_tier', 'fred', 'concentration'). Omit for all data."),
      // Params for specific sources:
      limit: z.number().optional().default(10)
        .describe("Limit results (for top_institutions, fee_index)"),
      quarters: z.number().optional().default(8)
        .describe("Number of quarters for trend data"),
      district: z.number().min(1).max(12).optional()
        .describe("Fed district number (for complaints, economic district view)"),
      charter: z.enum(["bank", "credit_union"]).optional()
        .describe("Charter type filter"),
      tiers: z.array(z.string()).optional()
        .describe("Asset size tier filter (for fee_index peer queries)"),
      top_n: z.number().optional().default(5)
        .describe("Top N for concentration analysis"),
    })
    ```

    **Tool description** (what Hamilton sees):
    "Query national summary data across all sources. Use 'source' to pick a data domain: call_reports (revenue trends, top institutions, tier/charter splits), economic (FRED rates, Beige Book themes, national summary), health (ROA, efficiency, deposits, loans, institution counts), complaints (CFPB district/institution summaries), fee_index (national/peer fee medians and distributions), derived (revenue concentration, fee dependency trends, per-institution revenue trends). Use 'view' to narrow to a specific aspect within the source."

    **Routing logic** -- switch on source, then switch on view within each:

    `call_reports`:
    - no view or "all": call getRevenueTrend(quarters), getTopRevenueInstitutions({limit, charter}), getRevenueByTier(quarters) -- return combined
    - "trend": getRevenueTrend(quarters)
    - "top_institutions": getTopRevenueInstitutions({limit, charter})
    - "by_tier": getRevenueByTier(quarters)
    - "by_district": getDistrictFeeRevenue(district)

    `economic`:
    - no view or "all": call getNationalEconomicSummary(), getBeigeBookThemes(), getFredSummary() -- return combined
    - "fred": getFredSummary()
    - "beige_book": getBeigeBookThemes()
    - "national": getNationalEconomicSummary()
    - "district": getDistrictEconomicSummary(district) -- require district param

    `health`:
    - no view or "all": call getIndustryHealthMetrics(), getHealthMetricsByCharter(), getDepositGrowthTrend(quarters), getLoanGrowthTrend(quarters), getInstitutionCountTrends(quarters)
    - "metrics": getIndustryHealthMetrics()
    - "by_charter": getHealthMetricsByCharter()
    - "deposits": getDepositGrowthTrend(quarters)
    - "loans": getLoanGrowthTrend(quarters)
    - "institution_counts": getInstitutionCountTrends(quarters)

    `complaints`:
    - require district param: getDistrictComplaintSummary(district)
    - If no district, return {error: "district parameter required for complaints source"}

    `fee_index`:
    - If charter or tiers provided: getPeerIndex({charter_type: charter, asset_tiers: tiers})
    - Otherwise: getIndexSnapshot(undefined, limit)

    `derived`:
    - no view or "all": call all three (getRevenueConcentration(top_n), getFeeDependencyTrend(quarters), getRevenuePerInstitutionTrend(quarters))
    - "concentration": getRevenueConcentration(top_n)
    - "dependency": getFeeDependencyTrend(quarters)
    - "revenue_per_institution": getRevenuePerInstitutionTrend(quarters)
