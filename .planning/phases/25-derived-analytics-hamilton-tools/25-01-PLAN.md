    Create `src/lib/crawler-db/derived-analytics.ts`. Per D-08 audit-first approach:

    **Audit result:** No existing function computes revenue concentration. `getFeeRevenueData` joins fees with financials but returns per-institution rows, not category-level aggregation. Gap confirmed.

    **Implementation (per D-04, D-05):**

    `getRevenueConcentration(topN = 5)`:
    1. Query: SELECT fee_category, SUM(amount) as total_fee_dollars, COUNT(DISTINCT crawl_target_id) as institution_count FROM extracted_fees WHERE fee_category IS NOT NULL AND amount > 0 AND review_status != 'rejected' GROUP BY fee_category ORDER BY total_fee_dollars DESC
    2. Compute total across all categories for percentage calculation
    3. Dollar volume axis: rank by total_fee_dollars, compute pct_of_total = (category_total / grand_total) * 100, cumulative for top N
    4. Query total distinct institutions: SELECT COUNT(DISTINCT crawl_target_id) FROM extracted_fees WHERE review_status != 'rejected'
    5. Prevalence axis: rank by institution_count, compute pct_of_institutions = (category_count / total_institutions) * 100
    6. Return both axes trimmed to topN, plus summary strings per D-05

    Interface:
    ```typescript
    export interface ConcentrationEntry {
      fee_category: string;
      value: number;        // total_fee_dollars or institution_count
      pct_of_total: number; // percentage of grand total
    }
    export interface RevenueConcentration {
      dollar_volume: ConcentrationEntry[];
      institution_prevalence: ConcentrationEntry[];
      summary: {
        top_n: number;
        dollar_volume_pct: number;   // e.g., 68.3
        prevalence_pct: number;      // e.g., 85.1
        total_fee_dollars: number;
        total_institutions: number;
      };
    }
    ```

    Use `sql` tagged template from connection.ts. Use `sql.unsafe()` only if dynamic ORDER BY needed (it is not -- hardcode DESC).

    - Edge: fewer than 2 quarters returns null for qoq; fewer than 5 returns null for yoy

    - YoY = (current - four_quarters_ago) / four_quarters_ago * 100

  <action>
    **Audit result per D-08:**
    - `getTierFeeRevenueSummary()` and `getCharterFeeRevenueSummary()` return latest-quarter snapshots. They satisfy the "position" part of DERIVE-02/03 but have NO time dimension.
    - `getRevenueIndexByDate(reportDate)` returns fee_income_ratio percentiles for a single date. Could be called per quarter to build trend.
    - Gap: No function returns fee dependency or revenue-per-institution as a time series with computed QoQ/YoY deltas.

    **Implementation:**

    `getFeeDependencyTrend(quarters = 4)`:
    1. Reuse `getTierFeeRevenueSummary()` and `getCharterFeeRevenueSummary()` for `current` (import from fee-revenue.ts)
    2. New SQL for trend: SELECT report_date, ROUND(AVG(CASE WHEN total_revenue > 0 THEN service_charge_income * 1.0 / total_revenue * 100 ELSE NULL END), 2) as avg_fee_income_ratio, COUNT(DISTINCT crawl_target_id) as institution_count FROM institution_financials WHERE fee_income_ratio IS NOT NULL GROUP BY report_date ORDER BY report_date DESC LIMIT ${quarters}
    3. Compute QoQ (index 0 vs index 1) and YoY (index 0 vs index matching ~4 quarters back)
    4. Return {current, trend, signals}

    `getRevenuePerInstitutionTrend(quarters = 4)`:
    1. Current: two queries -- AVG(service_charge_income * 1000) GROUP BY asset_size_tier, and GROUP BY charter_type, for latest report_date
    2. Trend: SELECT report_date, ROUND(AVG(service_charge_income * 1000), 0) as avg_sc_income, COUNT(DISTINCT crawl_target_id) as institution_count FROM institution_financials WHERE service_charge_income IS NOT NULL GROUP BY report_date ORDER BY report_date DESC LIMIT ${quarters}
    3. Note: service_charge_income is stored in thousands per v5.0 decision -- multiply by 1000 for display
    4. Compute QoQ/YoY signals same as above

    Helper function `computeTrendSignals(values: {date: string, value: number}[])` shared by both:
    - qoq: compare index 0 vs 1 (most recent vs previous quarter)
    - yoy: compare index 0 vs 4 (if exists)
    - Return nulls when insufficient data

    Interfaces:
    ```typescript
    export interface TrendSignals {
      qoq_change_pct: number | null;
      yoy_change_pct: number | null;
    }
    export interface FeeDependencyTrend {
      current: { by_tier: TierFeeRevenueSummary[]; by_charter: CharterFeeRevenueSummary[] };
      trend: { report_date: string; avg_fee_income_ratio: number; institution_count: number }[];
      signals: TrendSignals;
    }
    export interface RevenuePerInstitutionTrend {
      current: {
        by_tier: { tier: string; avg_sc_income: number; institution_count: number }[];
        by_charter: { charter: string; avg_sc_income: number; institution_count: number }[];
      };
      trend: { report_date: string; avg_sc_income: number; institution_count: number }[];
      signals: TrendSignals;
    }
    ```
