**Fixture: `seeded_db`**

Define a function-scoped fixture `seeded_db(test_db)` that:
1. Truncates `crawl_targets` via `test_db.execute("DELETE FROM crawl_targets")` + `test_db.commit()`
2. Yields `test_db`
3. On teardown: truncates again (cleanup after test)

This ensures every test starts with an empty crawl_targets regardless of session ordering.

**Test: `test_seed_fdic_populates_crawl_targets`**

Signature: `def test_seed_fdic_populates_crawl_targets(seeded_db, test_config):`

Mark: `@pytest.mark.e2e`

Steps:
1. Call `count = seed_fdic(seeded_db, test_config, limit=5)`
2. Assert `count >= 1` (API may return fewer if VT has <5 banks, but must be at least 1)
3. Query all rows: `rows = seeded_db.fetchall("SELECT * FROM crawl_targets")`
4. Assert `len(rows) >= 1`

Per D-03 — assert non-null fields for every row in a loop:
- `row["institution_name"]` is not None and not empty string
- `row["charter_type"]` is not None
- `row["asset_size"]` is not None
- `row["cert_number"]` is not None
- `row["source"]` is not None

Per D-04 — assert value ranges for every row:
- `row["charter_type"] == "bank"` (FDIC always inserts "bank")
- `row["source"] == "fdic"`
- `row["asset_size"] > 0`
- If `row["fed_district"]` is not None: `1 <= row["fed_district"] <= 12`

Per D-07 — assert website_url is populated:
- `row["website_url"]` is not None (FDIC WEBADDR is provided)

Note: FDIC API may normalize some URLs to None if WEBADDR field is empty. Assert website_url is not None only for FDIC rows that have a non-empty WEBADDR. To avoid flakiness, assert `sum(1 for r in rows if r["website_url"])` > 0 (at least one row has a URL) rather than asserting every row — some institutions may legitimately lack WEBADDR in the API response. Add a comment explaining this.

**Geography note:** `seed_fdic` in its current implementation does NOT filter by state — it pulls institutions sorted by asset size DESC. The `geography` fixture controls which geography the full pipeline uses in Phase 9. For Phase 2, seed tests pull from the national FDIC dataset with limit=5. This is correct per D-01.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test → FDIC API | seed_fdic calls api.fdic.gov over HTTPS — external network |
| test → NCUA ZIP | seed_ncua downloads ncua.gov ZIP over HTTPS — large payload (~20MB) |
| test → SQLite | Isolated file in tmp_path — no production DB path reachable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | FDIC API call in test | accept | Tests call real API with limit=5; rate limited by existing 0.5s crawl delay in test_config; acceptable for e2e marker tests |
| T-02-02 | Denial of Service | NCUA ZIP download | accept | 20MB download per test run; acceptable for e2e tests; test marked @pytest.mark.e2e so excluded from fast CI runs |
| T-02-03 | Information Disclosure | Test DB path via tmp_path | accept | pytest tmp_path is OS temp dir; session-scoped; deleted on session end; no sensitive data seeded |
| T-02-04 | Tampering | seeded_db truncates crawl_targets | mitigate | Fixture truncates ONLY crawl_targets; foreign-key constrained tables (crawl_results etc.) are empty at this stage so no cascade risk; prod_db_contamination_guard (autouse) prevents any write to data/crawler.db |
</threat_model>
