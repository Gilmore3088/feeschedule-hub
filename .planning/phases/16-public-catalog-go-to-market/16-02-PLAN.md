**1. Create `src/app/api/reports/catalog/route.ts` (GET, no auth)**

Returns all published reports where `is_public = true`, ordered by `published_at DESC`. Accepts optional query params `?type=national_index` and `?from=YYYY-MM-DD` for filtering. No authentication required — this is public data.

```typescript
export const dynamic = 'force-dynamic';

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const typeFilter = searchParams.get('type');
  const fromFilter = searchParams.get('from'); // ISO date string

  const sql = getSql();
  // Build dynamic WHERE — always include is_public=true
  // typeFilter: add AND report_type = ${typeFilter} if present and in VALID_REPORT_TYPES
  // fromFilter: add AND published_at >= ${fromFilter} if present and parseable as date
  const rows = await sql<PublishedReport[]>`
    SELECT id, report_type, slug, title, published_at
    FROM published_reports
    WHERE is_public = true
      ${typeFilter && VALID_TYPES.has(typeFilter) ? sql` AND report_type = ${typeFilter}` : sql``}
      ${fromDate ? sql` AND published_at >= ${fromDate}` : sql``}
    ORDER BY published_at DESC
    LIMIT 100
  `;
  return NextResponse.json({ reports: rows });
}
```

Validate `typeFilter` against `VALID_REPORT_TYPES` set before including in query (T-16-03). Parse `fromFilter` through `new Date(fromFilter)` and check `isNaN` before using.

**3. Create `src/app/api/reports/email-gate/route.ts` (POST, no auth)**

Accepts `{ email: string; slug: string }`. Returns `{ downloadUrl: string }` or error.

Validation:
- Email: must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` — return 400 if invalid (T-16-04)
- Slug: validate against `published_reports` table — return 404 if not found or not public (T-16-05)

Flow:
1. Validate inputs (T-16-04, T-16-05)
2. Fetch `artifact_key` from `published_reports JOIN report_jobs` WHERE slug matches and `is_public=true`
3. If `artifact_key` is null: return 202 `{ status: "pending", message: "Report PDF not yet available" }`
4. Store lead: `INSERT INTO report_leads (email, slug, requested_at) VALUES (${email}, ${slug}, now()) ON CONFLICT (email, slug) DO NOTHING` — this table may not exist yet; wrap in try/catch and log warning if it fails (do not block on lead capture failure)
5. Generate presigned URL via `generatePresignedUrl(artifact_key, 3600)` (1-hour TTL)
6. Return `{ downloadUrl }` with status 200

The `report_leads` table does not exist in the migration yet — the INSERT failing is acceptable behavior. Add a comment: `// TODO: Phase 17 — add report_leads migration`. The presigned URL generation must succeed even if lead storage fails.

Rate limiting: add `X-RateLimit` headers but do not enforce — this is a lead capture endpoint and blocking legitimate signups is worse than allowing duplicates for v2.0.

Per D-11: email gate for PDF. Per D-05: reads from published_reports table.
