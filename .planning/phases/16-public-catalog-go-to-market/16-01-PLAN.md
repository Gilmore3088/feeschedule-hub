Then add a JSON-LD Article script block as the **first child** of the returned `<main>` element (before the content div). Use `dangerouslySetInnerHTML` with a `<script type="application/ld+json">` tag. The JSON-LD object must be:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "How Bank Fee Index Works",
  "description": "A transparent account of how Bank Fee Index collects, classifies, and verifies fee data across 4,000+ financial institutions.",
  "url": "https://feeinsight.com/methodology",
  "datePublished": "2026-04-06T00:00:00Z",
  "author": {
    "@type": "Organization",
    "name": "Bank Fee Index",
    "url": "https://feeinsight.com"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Bank Fee Index",
    "url": "https://feeinsight.com"
  }
}
```

Build the JSON string via `JSON.stringify(jsonLdObject)` — do NOT use a raw string literal. The `SITE_URL` constant must be used for the url fields rather than hardcoded strings.

Do not modify any of the existing JSX content or styles. The `Section` component and all 6 sections remain unchanged. The only additions are: (1) the extended metadata export and (2) the JSON-LD script element.
