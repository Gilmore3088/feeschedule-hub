Create `src/lib/hamilton/home-data.ts` with a single exported async function `fetchHomeBriefingData()` that returns a typed `HomeBriefingData` object.

**Data assembly logic:**

1. **Thesis data**: Build a minimal `ThesisSummaryPayload` from live data:
   - Call `getNationalIndexCached()` to get IndexEntry[] (this is cheap, uses cache table).
   - Map the top 10 entries (already sorted by institution_count) into `top_categories` shape matching ThesisSummaryPayload.
   - Set `quarter` to current quarter string (e.g., "Q1 2026"), `total_institutions` from sum of unique institution counts.
   - Set `revenue_snapshot: null`, `fred_snapshot: null`, `beige_book_themes: []`, `derived_tensions: []` — the Home screen thesis is a lighter scope ("monthly_pulse") that does not require these heavy data sources. Per D-01 the thesis engine already handles lighter scopes.
   - Call `generateGlobalThesis({ scope: "monthly_pulse", data: thesisSummary })`.
   - Wrap in try/catch — on failure return `thesis: null` so the page can show empty state.

2. **Positioning evidence**: Filter `getNationalIndexCached()` result to spotlight categories only (using `getSpotlightCategories()`). For each, compute a simple percentile approximation: `percentile = ((amount - p25) / (p75 - p25)) * 50 + 25` clamped to 1-99. Return array of `{ feeCategory, displayName, medianAmount, p25, p75, institutionCount, maturityTier }`.

3. **Confidence level**: Derive from maturity tiers of the spotlight categories used in thesis:
   - All "strong" = "high"
   - Mix of strong/provisional = "medium"  
   - Any "insufficient" = "low"
