    **2. Create ManageBillingButton component:**
    Create `src/components/hamilton/settings/ManageBillingButton.tsx` as a `"use client"` component.

    Props interface:
    ```typescript
    interface ManageBillingButtonProps {
      hasStripeAccount: boolean;
      subscriptionStatus: "active" | "past_due" | "canceled" | "none";
      className?: string;
      style?: React.CSSProperties;
    }
    ```

    Implementation based on `subscriptionStatus`:
    - `"active"` -- render "Manage Billing" button, enabled, calls `createPortalSession()` via `startTransition`
    - `"past_due"` -- render "Update Payment" button with amber warning styling (`var(--hamilton-warning, #f59e0b)` border-left or background tint), calls `createPortalSession()`
    - `"canceled"` -- render "Reactivate" button with CTA styling (`burnished-cta` class), calls `createPortalSession()`
    - `"none"` (or `hasStripeAccount === false`) -- render disabled button with text "Subscribe to manage billing", no action. Alternatively render an `<a href="/subscribe">` styled link with text "Subscribe to Pro"

    Common behavior:
    - Use `useTransition` from React for the pending state (server action with redirect)
    - Use `useState` for error message
    - Loading state: button text changes to "Opening..." and button is disabled
    - Error state: inline `<p>` below the button with text "Unable to open billing portal. Please try again." in `var(--hamilton-error, #ba1a1a)` color, `0.75rem` font size
    - Clear error on new click attempt

    Follow the exact pattern from RESEARCH.md Code Examples "ManageBillingButton Component" section, which uses Hamilton design tokens for error color.
