    **1. Monitor main grid (page.tsx lines ~65-72):**

    Current (broken at 768px):
    ```tsx
    <div style={{ display: "grid", gridTemplateColumns: "7fr 5fr", gap: "3rem" }}>
    ```

    Convert to:
    - Add `@container` class to the `<main>` element (the outermost wrapper). Keep existing inline styles on `<main>` (minHeight, backgroundColor) -- `@container` is compatible with these.
    - Replace the inline `style={{ display: "grid", gridTemplateColumns: "7fr 5fr", gap: "3rem" }}` with Tailwind classes: `className="grid grid-cols-1 gap-8 @3xl:grid-cols-[7fr_5fr] @3xl:gap-12"`

    - Use `@3xl` rather than `@2xl` (672px) because the Monitor content (SignalFeed cards with 2rem padding) needs at least ~750px to render the two-column layout without cramping

    **2. StatusStrip (StatusStrip.tsx):**

    The StatusStrip renders 5-6 metrics in a horizontal flex row with `justify-content: space-between`. At narrow widths this overflows.

    - Wrap the StatusStrip root element with `@container` class
    - Convert the metrics flex row from a single horizontal line to a responsive grid: `className="grid grid-cols-2 gap-4 @xl:grid-cols-3 @2xl:flex @2xl:justify-between @2xl:items-center"`
    - At narrow: 2-column grid of metric items
    - At `@xl` (576px): 3-column grid
    - At `@2xl` (672px): original horizontal flex layout
    - Remove inline `style` for display/justifyContent/alignItems on the metrics container and replace with the Tailwind classes above
    - Keep 1px divider elements but hide them at narrow widths: add `className="hidden @2xl:block"` to divider elements (they only make sense in horizontal layout)

    **Important per D-05:** Use `@container` / `@xl:` / `@2xl:` / `@3xl:` container query variants (NOT viewport `md:` / `lg:` breakpoints). Components adapt to their parent container width.
