  <action>
    Create `src/lib/report-engine/freshness.ts`.

    Export `FreshnessResult` interface (can be in this file or types.ts — keep it in freshness.ts to avoid circular imports).

    Export `checkFreshness(scope: 'national' | 'state' | 'peer', stateCode?: string): Promise<FreshnessResult>`.

    Postgres does not have a built-in MEDIAN aggregate. Use PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY ...) — the standard SQL median for Postgres/Supabase:
    ```sql
    SELECT PERCENTILE_CONT(0.5) WITHIN GROUP (
      ORDER BY EXTRACT(EPOCH FROM (NOW() - last_crawled_at)) / 86400.0
    ) AS median_age
    FROM crawl_targets
    WHERE last_crawled_at IS NOT NULL
    ```
    Add `AND state_abbr = ${stateCode}` when scope is 'state'.

    National threshold: 120. State threshold: 90. Peer: 120.

    If the query returns null (no crawl_targets with timestamps), treat as age = 999 days (stale — fail safe).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API routes → report_jobs | report_type and params come from API caller; must be validated before INSERT |
| freshness gate → crawl_targets | Read-only query; no injection risk but null-result must fail safe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | report_jobs.status | mitigate | Status transitions enforced via CHECK constraint in migration; application code validates before UPDATE |
| T-13-02 | Information Disclosure | data_manifest JSONB | accept | Manifest contains query shapes + row counts, not row data; low sensitivity |
| T-13-03 | Denial of Service | checkFreshness() | accept | Read-only query on indexed column; single PERCENTILE_CONT is O(n log n) but run < 1/minute |
| T-13-04 | Spoofing | report_jobs.user_id | mitigate | user_id populated from verified session in API route, never from request body |
</threat_model>
