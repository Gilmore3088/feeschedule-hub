    async def generate_report(job_id: str, html: str, report_type: str) -> dict:
        """Render assembled HTML to PDF, upload to R2, update job status."""
        from fee_crawler.workers.report_render import render_and_store, update_job_status

            raise

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API caller → generate_report Modal | html parameter could be arbitrarily large; job_id must be a valid UUID |
| Modal worker → R2 | PDF bytes upload; credentials in Modal secrets, never in code |
| Editor Claude response → approved flag | JSON parse failure must default to approved=false (fail safe) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05 | Tampering | generate_report job_id param | mitigate | Only Next.js API routes (authenticated) call generate_report; job_id validated as UUID before DB update |
| T-13-06 | Information Disclosure | R2 artifact_key stored in Supabase | accept | Key is an opaque path, not a signed URL; actual download requires separate presign step with auth check |
| T-13-07 | Denial of Service | Large HTML payload to Modal | accept | Modal has memory=2048 limit; Playwright will OOM before resource exhaustion; report HTML is bounded by template structure |
| T-13-08 | Elevation of Privilege | Editor JSON parse failure | mitigate | JSON.parse exceptions caught; on parse error default to approved=false + flaggedSections=[{severity:'major', reason:'Editor response unparseable'}] |
| T-13-09 | Repudiation | R2 upload without job record | mitigate | update_job_status('rendering') is called BEFORE render_and_store(); if upload fails, job reaches 'failed' state with error logged |
</threat_model>
