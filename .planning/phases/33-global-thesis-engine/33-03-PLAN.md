// Phase 33: Generate global thesis before sections (per D-01, D-04)
// Thesis uses condensed payload (~5KB) not full payload.

let thesis: ThesisOutput | null = null;
try {

  thesis = await generateGlobalThesis({ scope: 'quarterly', data: summary });

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ThesisOutput.narrative_summary → section context string | Thesis narrative is concatenated into user message sent to Claude for section generation |
| Claude thesis output → section prompt | If thesis output contains adversarial text, it could influence section generation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-07 | Tampering | narrative_summary injected into section prompt | mitigate | narrative_summary is generated by the same Claude model under the same HAMILTON_VOICE system prompt (not user-controlled); risk is model self-injection which is low; narrative is labeled "GLOBAL THESIS" so the section model treats it as context, not instruction |
| T-33-08 | Denial of Service | thesis generation adds latency before 6 parallel section calls | accept | Thesis adds ~1 API call upfront; sections still run in parallel after; total wall time increase is bounded by 1 extra Claude call (~10-20s); acceptable for report generation use case |
| T-33-09 | Availability | thesis generation failure cascades to failed report | mitigate | try/catch around entire thesis block; on any error, thesis is null and thesisContext is empty string; sections degrade gracefully to no-thesis mode |
</threat_model>
