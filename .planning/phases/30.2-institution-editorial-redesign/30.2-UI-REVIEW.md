## Top 3 Priority Fixes

1. **Rating card tint opacity is 100% instead of /30** — The yellow rating card renders a solid `#fff9c4` background, making it visually heavier than the mockup's subtle 30% tint (`bg-[#fff9c4]/30`). Same issue applies to green and red variants. The RATING_CONFIG stores raw hex values which are then injected via `style={{ backgroundColor: ratingConfig.bg }}`, bypassing Tailwind's opacity modifier. Fix by using CSS rgba: change `"#fff9c4"` to `"rgba(255,249,196,0.30)"`, `"#f0f9f0"` to `"rgba(240,249,240,0.30)"`, and `"#ffebee"` to `"rgba(255,235,238,0.30)"` in the `RATING_CONFIG` object at `page.tsx:76-78`.

2. **Wire Transfers group missing primary (accent-border) treatment** — The mockup explicitly applies `border-l-4 border-primary` to both the Overdraft & NSF group AND the Wire Transfers group (code.html:289). The implementation defines `PRIMARY_GROUPS` as only `["Overdraft & NSF"]` (page.tsx:58), so Wire Transfers renders without the accent left-border. Add `"Wire Transfers"` to `PRIMARY_GROUPS` and consider whether it should also open by default.

3. **Column header row renders on all fee groups, not just the first open one** — The mockup shows "Market Distribution / Amount / Freq." column headers only in the Overdraft & NSF group summary (code.html:212-215). Non-primary groups in the mockup have no column header row in their `<summary>`. The implementation places the header row in every group's `<summary>` regardless of `isPrimary` (fee-group.tsx:69-73). This adds visual noise to collapsed secondary groups and doesn't match the mockup. Wrap the `hidden md:flex gap-12` column header div in `{isPrimary && ...}`.

### Pillar 3: Color (3/4)

**Passing:**
- Base parchment: `bg-[#fbf9f4]` — matches `#fbf9f4` from design system, NOT the wrong `#FAF7F2` — decision D-02 satisfied.
- `surface-container-low` (#f5f3ee) used consistently for stat cards, Stat component, footer hover states, CTA section.
- Primary `#8a4c27` (burnt sienna) used correctly — not `#C44B2E` terracotta.
- `outline-variant` `#d8c2b8` used at low opacity (10-30%) for ghost borders in tables and footer — compliant with design system's "Ghost Border Fallback" rule.
- CTA button uses `bg-gradient-to-br from-[#8a4c27] to-[#a7633c]` — matches the "Burnished CTA" specification from DESIGN.md Section 2.

**Issues:**
- **Rating card background opacity issue** (page.tsx:379): `backgroundColor: ratingConfig.bg` injects `#fff9c4` at 100% opacity. The mockup applies `bg-[#fff9c4]/30` (30% opacity) via Tailwind. The fully-opaque version creates a harder, less parchment-consistent look. All three rating colors are affected.
- Mini CTA background uses `bg-[#e4e2dd]/50` (page.tsx:485). The mockup uses `bg-surface-container-highest/50` which maps to `#e4e2dd/50` — this is numerically correct. However, the mockup also adds `border border-outline-variant/20` to the mini CTA card (code.html:188), which the implementation omits. This is a No-Line Rule grey area — the border is at 20% opacity, which the design system explicitly permits as a "Ghost Border Fallback." Its omission makes the mini CTA card slightly less defined.
- The green/red rating icon colors (`#4caf50`, `#f44336`) are Material Design green/red, not from the parchment palette. These are reasonable semantic choices for the rating system and align with typical pass/fail conventions, but they introduce colors outside the defined palette. The design system doesn't define semantic status colors, so this is acceptable.

### Pillar 6: Experience Design (3/4)

**Passing:**
- `notFound()` guard on invalid institution ID (page.tsx:213) and missing institution (page.tsx:219) — both paths handled.
- `try/catch` on `getRelatedReports()` (page.tsx:201) and `generateStaticParams()` (page.tsx:109) — graceful degradation.
- Empty fee groups: `if (rows.length === 0) return null` (page.tsx:519) and `if (fees.length === 0) return null` (fee-group.tsx:48) — no empty sections rendered.
- Fee amount null/zero states: `fee.amount === null` → italic "N/A", `fee.amount === 0` → "$0.00" (fee-group.tsx:117-121) — both edge cases handled.
- Empty related reports: "No peer reports published yet for this segment." fallback text (page.tsx:724).
- Conditional pro sections: all Pro content gated with `isPro &&` checks and minimum data thresholds (e.g., `scorecardComparisons.length >= 2`, `distributionData.length >= 2`).
- Market distribution bars gracefully degrade: shows a dim empty bar when no index entry available (fee-group.tsx:110).

**Issues:**
- **No loading skeleton for the institution page.** The page is `export const dynamic = "force-dynamic"`, meaning it runs on every request without ISR. If the DB is slow, the user sees a blank white screen until the full server render completes. A `loading.tsx` file in the `[id]` directory would provide an instant skeleton. This is the most significant UX gap.
- **No Suspense boundaries around Pro sections.** Pro data fetches (market concentration, revenue trend, peer ranking, distribution data) are all awaited in serial/parallel via `Promise.all` at the top of the page function. A slow call-report query blocks the entire page. Wrapping Pro sections in `<Suspense>` would allow the public content to stream immediately.
- **No error boundary for the page-level try/catch pattern.** If `getInstitutionById` or `getFeesByInstitution` throws (DB connection error), the page throws an unhandled error. The `notFound()` guards only handle null returns, not exceptions. Consider wrapping the primary data fetch in try/catch with a redirect to a generic error page.
