# Plan 12-01: Hamilton Voice, generateSection API, and Numeric Validator

## Objective

Create the Hamilton AI analyst module: a versioned voice definition, a typed `generateSection()` function that calls Claude with strict data grounding, and a post-generation numeric validator that confirms zero invented statistics before any section is finalized.

## Context

- CONTEXT.md D-01 through D-05: McKinsey Partner tone, narrative-first, locked in `src/lib/hamilton/voice.ts`
- CONTEXT.md D-17, D-18: Numeric validator cross-checks every number in Hamilton's output against source data JSON
- Existing: `src/lib/brief-generator.ts` (HTML generation foundation)
- Existing: `@anthropic-ai/sdk` already in package.json
- No `src/lib/hamilton/` directory exists yet

## Tasks

### Task 1: Create Hamilton module scaffold and voice definition
**Type:** auto

Create `src/lib/hamilton/` with:
- `voice.ts` — versioned persona definition with 6 concrete stylistic rules and forbidden patterns
- `types.ts` — TypeScript types for section generation (SectionType, SectionInput, SectionOutput, ValidatedSection)
- `index.ts` — barrel export

Voice rules must encode D-01 through D-05 as concrete, checkable directives:
1. Always third-person narrative ("Our analysis shows" allowed; "I think" forbidden)
2. Every statistic must be followed by its source datum key
3. Section title states conclusion not topic ("Montana overdraft fees run 23% above national" not "Overdraft Fee Analysis")
4. Forbidden words: might, could potentially, perhaps, interesting, very, really, quite + all casual language
5. Sentence structure: situation → complication → finding → implication (not finding → data → hedging)
6. Numbers formatted consistently: currency with $ and comma separators, percentages to one decimal

**Done:** `src/lib/hamilton/voice.ts` exports `HAMILTON_VOICE` constant with version, persona, rules[], forbidden[], tone, and system prompt. `types.ts` exports SectionType union, SectionInput, SectionOutput, ValidatedSection.

### Task 2: Implement generateSection() with Claude integration
**Type:** auto

Create `src/lib/hamilton/generate.ts` implementing:

```typescript
export async function generateSection(input: SectionInput): Promise<SectionOutput>
```

- Takes `SectionInput`: `{ type: SectionType, title: string, data: Record<string, unknown>, context?: string }`
- Builds system prompt from `HAMILTON_VOICE.systemPrompt`
- Appends strict instruction: "Use only the figures provided in the DATA block below. Do not invent, estimate, or extrapolate any statistic not present."
- Formats data as labeled JSON block in the user message
- Calls `@anthropic-ai/sdk` with `claude-sonnet-4-5` (latest available)
- Returns `SectionOutput`: `{ narrative: string, wordCount: number, model: string, usage: { inputTokens, outputTokens } }`
- Handles API errors explicitly with meaningful messages (D-05 compliance: no silent failures)

**Done:** `generateSection()` returns structured output. Calling with an empty data object and a section type produces a narrative. Function throws on API error with context.

### Task 3: Implement numeric validator
**Type:** auto

Create `src/lib/hamilton/validate.ts` implementing:

```typescript
export function validateNumerics(narrative: string, sourceData: Record<string, unknown>): ValidationResult
```

Algorithm:
1. Extract all numeric tokens from narrative text using regex: integers, decimals, percentages, currency amounts
2. Flatten `sourceData` recursively to collect all numeric leaf values
3. For each narrative number: check if it appears in source values (with tolerance for rounding: ±0.5 for integers, ±0.005 for percentages displayed to 1 decimal)
4. Return `ValidationResult`: `{ passed: boolean, inventedNumbers: string[], checkedCount: number, sourceValues: number[] }`

Export `validateSection(output: SectionOutput, input: SectionInput): ValidatedSection` which composes generateSection validation and attaches result to output.

**Done:** `validateNumerics("The median fee is $12.50 and the rate is 23.4%", { median: 12.50, rate: 23.4 })` returns `{ passed: true }`. `validateNumerics("The fee is $99.00", { median: 12.50 })` returns `{ passed: false, inventedNumbers: ["99.00"] }`.
