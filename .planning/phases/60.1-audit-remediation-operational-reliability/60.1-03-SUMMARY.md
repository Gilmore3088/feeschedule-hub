# Phase 60.1 Plan 03: Report Job Failure Propagation Summary

**One-liner:** `markJobFailed()` helper wired to all three Modal trigger failure paths so no report_jobs row is left stuck in 'pending' after a trigger error.

## What Was Built

Resolved audit Finding 3 (P1): the ghost-queue scenario where `POST /api/reports/generate` inserts a `report_jobs` row in `pending` status, then silently drops trigger errors inside `after(...)`, leaving jobs permanently stuck.

### Changes to `src/app/api/reports/generate/route.ts`

**Added `markJobFailed(jobId, reason)` helper (lines 34-47):**
- Issues `UPDATE report_jobs SET status='failed', error=..., completed_at=now() WHERE id=...`
- Truncates reason to 490 chars (fitting within the column with the `modal trigger failed: ` prefix)
- Wraps its own DB call in try/catch so a secondary DB failure cannot crash the outer `after()` closure
- Uses schema column `error` (confirmed via `supabase/migrations/20260406_report_jobs.sql`) — plan spec referenced `error_message` but the actual column is `error`

**Refactored Modal trigger block (lines 156-187):**

| Path | Before | After |
|------|--------|-------|
| `MODAL_REPORT_URL` missing | `console.warn(...)`, return 202 | `markJobFailed(...)`, return 503 |
| Modal returns non-OK status | `console.error(status)`, job stays pending | `markJobFailed(jobId, 'modal http {status}')` |
| `fetch` throws | `console.error(message)`, job stays pending | `markJobFailed(jobId, reason)` |

Auth, validation, freshness gate, INSERT logic, and success-path response format are unchanged.

## Deviations from Plan

### Schema column name mismatch

**Rule 1 - deviation from plan spec, not a bug introduced by this plan.**

- **Found during:** Task 1 (read_first step - inspecting migrations)
- **Issue:** Plan spec referenced `error_message` as the failure column; actual schema column is `error` (defined in `supabase/migrations/20260406_report_jobs.sql`)
- **Fix:** Used `error` in the SET clause instead of `error_message`. Also used `completed_at` (present in schema) instead of `updated_at` (not in schema).
- **Files modified:** `src/app/api/reports/generate/route.ts`
- **Commit:** 5afe327
