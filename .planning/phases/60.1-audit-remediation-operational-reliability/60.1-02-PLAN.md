       ```python
       def require_postgres(reason: str) -> None:
           """Raise if DATABASE_URL is not configured.

           Pipeline tables (jobs, platform_registry, cms_confidence,
           document_r2_key, document_type_detected, doc_classification_confidence)

           """

               raise RuntimeError(
                   "SQLite not supported for pipeline tables; set DATABASE_URL "
                   f"to a Postgres connection string. Reason: {reason}. "
                   "See .planning/phases/60.1-audit-remediation-operational-reliability/ "
                   "for the schema authority decision."
               )
       ```

Pipeline tables (jobs, platform_registry, cms_*, document_*) require Postgres via DATABASE_URL. Use `require_postgres(...)` at every call site that touches pipeline tables."

    4. In the `Database.__init__` (or equivalent SQLite bootstrap function), if `_DATABASE_URL` is unset AND the caller path will need pipeline tables, do NOT auto-fail — leave legacy SQLite usage working for the legacy tables. Add a clear comment above the SQLite bootstrap section: `# LEGACY: SQLite path covers crawl_targets/crawl_runs/crawl_results/extracted_fees only.
