## Accomplishments

- Added `SubprocessFailed(RuntimeError)` with `cmd`, `returncode`, `stdout_tail`, `stderr_tail` attributes — exception message embeds both tails for Modal UI visibility
- Added `run_checked()` helper using `_subprocess` alias so bare `subprocess.run` is banished from all call sites
- Refactored `run_pdf_extraction`, `run_browser_extraction`, `run_post_processing` — first failure now raises and aborts the function
- Refactored `ingest_data` — each ingestor runs to completion, any failures collected, aggregate `SubprocessFailed` raised at end (explicit opt-in graceful continuation per CONTEXT.md LOCKED decision)

## Decisions Made

- Used `_subprocess` as a module-level alias for `subprocess` so the AST verification check (`n.func.value.id in ('subprocess', '_subprocess')`) can trivially distinguish the helper-internal call from any errant bare call sites added in future
- `ingest_data` uses try/except SubprocessFailed per loop iteration to preserve graceful continuation, then raises a synthetic aggregate `SubprocessFailed` at function end — this is the explicit opt-in required by the LOCKED decision in CONTEXT.md
- Tests use `sys.executable` and `-c` inline scripts for portability (no `/bin/false` dependency)
