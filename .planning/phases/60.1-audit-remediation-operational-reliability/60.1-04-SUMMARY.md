# Phase 60.1 Plan 04: CI/Modal Python Dependency Unification Summary

**One-liner:** CI now installs `fee_crawler/requirements.txt` (the Modal manifest) in both jobs; root `requirements.txt` is a one-line `-r` redirect eliminating manifest drift structurally.

## What Was Done

### Task 1: Unify CI and root requirements on the Modal manifest

**Problem (Finding 4, P1):** `.github/workflows/e2e-tests.yml` installed the root `requirements.txt`, which was missing seven packages that Modal images install from `fee_crawler/requirements.txt` — including `psycopg2-binary`, `asyncpg`, `httpx`, `anthropic`, `playwright`, `playwright-stealth`, and `boto3`. This meant CI could pass while Modal deployments failed on import errors.

**Changes made:**

1. **`.github/workflows/e2e-tests.yml`** — Both `e2e-fast` and `e2e-full` job "Install Python dependencies" steps updated from `pip install -r requirements.txt` to `pip install -r fee_crawler/requirements.txt`. All other steps (system deps, pytest invocation, env vars) left unchanged.

2. **`fee_crawler/requirements.txt`** — Added two packages that were in the root manifest but absent from the Modal manifest:
   - `python-dotenv>=1.0` — used by `fee_crawler/__main__.py` (`load_dotenv`) but missing from Modal manifest
   - `playwright-stealth>=2.0.3` — already present in the worktree's canonical manifest; confirmed it belongs here

3. **`requirements.txt`** — Rewritten from a 7-line explicit list to a single-line redirect: `-r fee_crawler/requirements.txt`. This preserves `pip install -r requirements.txt` muscle memory for developers and any external tooling while routing to the single source of truth.
