<decisions>
## Implementation Decisions

### Finding 1 — Modal subprocess failures (P0)
- LOCKED: Introduce a shared subprocess helper in `fee_crawler/modal_app.py` (or a sibling module) that wraps `subprocess.run(..., check=True)` and raises on non-zero exit codes.
- LOCKED: Raised exceptions MUST include tails of stdout and stderr so Modal UI shows cause.
- LOCKED: Refactor `run_post_processing`, `run_daily_report`, and all other scheduled Modal functions touched in the audit to use the helper. No scheduled job may swallow subprocess failure.
- LOCKED: If a step is intentionally allowed to degrade, it must opt-in explicitly (named flag), not by default.

### Finding 2 — Schema authority (P0)
- LOCKED: Establish one authoritative schema source for crawler pipeline tables (`jobs`, `platform_registry`, `cms_confidence`, `document_r2_key`, `document_type_detected`, `doc_classification_confidence`).
- DECISION TO MAKE (planner): Choose between (a) bring SQLite bootstrap in `fee_crawler/db.py` to parity with Postgres/Supabase migrations, OR (b) formally retire SQLite for pipeline paths with a loud error. Project memory indicates "Kill SQLite" is a stated direction — prefer option (b) unless local dev ergonomics require option (a).
- LOCKED: `modal_preflight.py` must exercise the same structural assumptions as production.

### Finding 3 — Stuck report jobs (P1)
- LOCKED: `src/app/api/reports/generate/route.ts` must transition the `report_jobs` row to `failed` whenever Modal triggering fails (non-OK response, thrown error, or missing `MODAL_REPORT_URL`).
- LOCKED: Missing `MODAL_REPORT_URL` is a terminal error for that job, not a silent log line.
- LOCKED: Status update must happen even when the trigger is launched from `after(...)`.

### Finding 4 — CI/Modal dependency parity (P1)
- LOCKED: CI (`.github/workflows/e2e-tests.yml`) must install the same Python environment used by Modal.
- DECISION TO MAKE (planner): Either (a) delete root `requirements.txt` and have CI install `fee_crawler/requirements.txt`, OR (b) unify both manifests to a single source. Prefer option (a) unless root `requirements.txt` has non-crawler consumers.

### Finding 5 — FFIEC scaling contract (P1)
- LOCKED: Restore a single shared scaling helper (reinstate `_apply_ffiec_scaling` or equivalent).
- LOCKED: `ingest_call_reports.py`, `test_call_report_scaling.py`, and `scripts/migrations/023-fix-ffiec-scaling.sql` must agree on unit direction. The migration and tests say FFIEC arrives in thousands and must be multiplied up to dollars — adopt that as the canonical contract.
- LOCKED: Before backfilling stored values, verify direction against a sample of production rows. Backfill is a follow-up concern; this phase must leave the *contract* consistent even if historical data remains wrong.
- LOCKED: Update `migration docs` / comments to match.

### Finding 6 — Monthly pulse (P2)
- LOCKED: `run_monthly_pulse` must use `BFI_APP_URL`, not `NEXT_PUBLIC_APP_URL`.
- DECISION TO MAKE (planner): Either give the function a cron schedule OR explicitly document it as manual-only (rewrite docstring and remove scheduling-like language). Default to "manual-only" unless user indicates otherwise during planning.

### Phase 59 test regressions
- LOCKED: `fee_crawler/tests/test_stealth_fetcher.py` must patch the module-level `Stealth` symbol. Fix the import in `fee_crawler/pipeline/playwright_fetcher.py` to expose `Stealth` at module scope so the existing patch target becomes valid.
- LOCKED: `fee_crawler/tests/test_call_report_scaling.py` must collect without error. The fix here is coupled to Finding 5 (helper restoration).

### Acceptance — pytest & vitest
- LOCKED: `pytest -m "not e2e and not llm and not slow"` must exit green with no collection errors.
- LOCKED: `npx vitest run` must remain green (387 passing pre-change).
- LOCKED: No net new lint errors introduced by this phase.

### Claude's Discretion
- Exact module layout for the subprocess helper (new file vs in `modal_app.py`).
- Internal refactor of `report_jobs` status update (direct SQL vs existing helper).
- Test structure for the subprocess helper (unit test + at least one regression test per Modal function touched).
- Whether to delete or retain root `requirements.txt` after consolidation.

</decisions>

<specifics>
## Specific Ideas

- Shared subprocess helper signature suggestion: `run_checked(cmd: list[str], *, cwd=None, env=None, tail_lines: int = 40) -> subprocess.CompletedProcess` that raises `SubprocessFailed(cmd, returncode, stdout_tail, stderr_tail)` on non-zero.
- Report job failure path should set `status='failed'`, `error_message='modal trigger failed: <reason>'`, `updated_at=now()`.
- For the schema authority, Supabase migrations (`supabase/migrations/`) are already the Postgres source of truth — align SQLite bootstrap to reflect that contract OR delete SQLite pipeline support and raise a clear `RuntimeError("SQLite not supported for pipeline tables; use DATABASE_URL")` when pipeline code is invoked under SQLite.
- Validation commands to run at end of phase:
  - `pytest -m "not e2e and not llm and not slow" -q`
  - `npx vitest run`
  - Manual probe: import `_apply_ffiec_scaling` from `fee_crawler.commands.ingest_call_reports`
  - Manual probe: `from fee_crawler.pipeline import playwright_fetcher; assert hasattr(playwright_fetcher, 'Stealth')`

</specifics>
