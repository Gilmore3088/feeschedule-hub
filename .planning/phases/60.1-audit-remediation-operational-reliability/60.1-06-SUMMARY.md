# Phase 60.1 Plan 06: Hoist Stealth Import to Module Scope Summary

**One-liner:** Module-level `Stealth` import with `ImportError` fallback restores `unittest.mock.patch` target and closes the Phase 59 test regression in `test_stealth_fetcher.py`.

## What Was Built

`fee_crawler/pipeline/playwright_fetcher.py` previously imported `Stealth` from `playwright_stealth` inside the `fetch_with_browser` function body (around line 364). The test file `fee_crawler/tests/test_stealth_fetcher.py` patches the symbol via `"fee_crawler.pipeline.playwright_fetcher.Stealth"`, which requires the name to exist at module scope — it did not, causing `AttributeError` on test collection.

**Fix applied:**

1. Added a `try/except ImportError` block immediately after the standard library imports at the top of the module:
   ```python
   try:
       from playwright_stealth import Stealth  # noqa: F401  (used at runtime + patched in tests)
   except ImportError:  # pragma: no cover
       Stealth = None
   ```

2. Replaced the in-function `from playwright_stealth import Stealth` / `_stealth = Stealth()` with a `None`-guard:
   ```python
   if Stealth is None:
       raise RuntimeError(
           "playwright_stealth is not installed; install it via "
           "fee_crawler/requirements.txt to enable stealth fetching"
       )
   _stealth = Stealth()
   ```

Runtime behavior is unchanged when `playwright_stealth` is installed (Modal image pulls it from `fee_crawler/requirements.txt`). When missing, a clear `RuntimeError` is raised rather than an `ImportError` buried inside the function.
