<objective>
Fix the Call Report thousands-scaling bug so all dollar-denominated financial data is returned in actual dollars, not in FDIC/NCUA thousands convention.

Purpose: Reports currently show $0.0B for industry service charges because the raw thousands values are divided by 1B in formatters. After this fix, $48.2B appears correctly. Every downstream consumer (assemblers, hero cards, Hamilton) gets correct dollar amounts without changes.

Output: Patched financial.ts and call-reports.ts with `* 1000` on dollar columns, plus unit tests proving the fix.
</objective>
