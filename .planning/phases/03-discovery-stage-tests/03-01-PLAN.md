FIXTURE — discovered_db (module-scoped):
- Scope: `module` — discovery involves real HTTP requests (30-120s/institution); re-running per test is wasteful and flaky
- Setup: call `test_db.execute("DELETE FROM discovery_cache")` then `test_db.execute("DELETE FROM crawl_targets")` and `test_db.commit()` before seeding
- Seed: `count = seed_fdic(test_db, test_config, limit=3)` — FDIC only because NCUA rows have website_url=NULL (nothing to discover)
- Skip guard: `if count == 0: pytest.skip("seed_fdic returned 0 rows — FDIC API unavailable")` so the test suite degrades gracefully when the FDIC API is down
- Filter institutions: `targets = test_db.fetchall("SELECT id, institution_name, website_url, state_code, asset_size FROM crawl_targets WHERE website_url IS NOT NULL")` — only institutions with a URL are discoverable
- Skip guard: `if not targets: pytest.skip("No FDIC institutions have website_url — cannot run discovery tests")`
- Discovery loop: for each target, call `_discover_one(dict(target), test_config, concurrent=False, force=False)` wrapped in a 60-second timeout (use `concurrent.futures.ThreadPoolExecutor(max_workers=1)` with `future.result(timeout=60)` — works cross-platform unlike signal.alarm which is UNIX-only). On `TimeoutError` or any exception, print a warning and continue.
- After the loop: `test_db.commit()` — in case the thread-local writer did not auto-commit (belt-and-suspenders)
- Yield: `yield test_db`
- Teardown: DELETE FROM discovery_cache, DELETE FROM crawl_targets, commit

THREAD-LOCAL DB NOTE: `_discover_one` calls `get_worker_db(config)` which is thread-local. When called from the main thread (non-concurrent), it creates a separate SQLite connection to the SAME file as test_db. Both connections write to the same WAL file. After the loop, re-read results from test_db (fresh SELECT) — do NOT rely on test_db's in-memory cache to see the worker's writes; issue a `test_db.conn.execute("SELECT 1")` to force a read-consistent view, or just do the SELECT query fresh.

| test process -> external websites | UrlDiscoverer makes real HTTP requests to institution websites; no user data crosses this boundary, only crawl traffic |
| test DB file -> worker thread | _discover_one uses thread-local get_worker_db; both connect to the same tmp SQLite file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | _discover_one HTTP requests | mitigate | 60s timeout via concurrent.futures future.result(timeout=60) prevents test hanging on unresponsive sites |
| T-03-02 | Denial of Service | FDIC API calls in fixture | accept | Limit=3 caps API requests; skip guard handles API unavailability gracefully |
| T-03-03 | Information Disclosure | test SQLite file in tmp_path | accept | tmp_path is OS temp dir; no PII in test data; file deleted at session end |
| T-03-04 | Tampering | prod_db_contamination_guard | mitigate | Session-autouse fixture in conftest.py asserts data/crawler.db mtime unchanged; already implemented in Phase 1 |
