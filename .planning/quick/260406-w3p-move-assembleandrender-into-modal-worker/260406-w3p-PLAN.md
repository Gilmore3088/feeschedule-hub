<objective>
Move the assembleAndRender() pipeline out of Vercel's serverless function lifecycle and into Modal.

Currently, the generate route fires off assembleAndRender() as a non-awaited async closure
that gets killed when Vercel reclaims the function instance. The fix: Vercel only inserts the
job row and sends a lightweight trigger to Modal. Modal calls back to a new internal API
endpoint that runs the TS assembly logic, gets the HTML, then does PDF + R2.

Purpose: Report generation reliably completes instead of dying on Vercel serverless timeout.
Output: Three files modified so the full pipeline runs on Modal with a 5-minute timeout.
</objective>

  <name>Task 1: Create internal /api/reports/[id]/assemble endpoint</name>
  <files>src/app/api/reports/[id]/assemble/route.ts</files>
  <action>
Create a new POST endpoint at `src/app/api/reports/[id]/assemble/route.ts`.

This endpoint is called by Modal (server-to-server), NOT by browsers. Auth via a shared
secret header `X-Internal-Secret` checked against env var `REPORT_INTERNAL_SECRET`.
This prevents external callers from triggering expensive assembly.

Logic:
1. Validate `X-Internal-Secret` header matches `process.env.REPORT_INTERNAL_SECRET`.
   Return 401 if missing/mismatched. Guard against empty string (length > 0 check).
2. Await `params` to get `id` (Next.js 15 pattern — params is a Promise).
3. Read the job row from `report_jobs` WHERE id = jobId AND status = 'pending'.
   If not found or not pending, return 404 with `{ error: 'Job not found or not pending' }`.
4. Extract `report_type` and `params` (JSON parse params column) from the job row.
5. Call `assembleAndRender(reportType, parsedParams, jobId)` — this already handles
   status updates to 'assembling' and 'failed' internally.
6. Return `{ html }` with status 200.
7. On error from assembleAndRender, return 500 with `{ error: message }`.
   assembleAndRender already marks the job as 'failed', so no extra DB write needed.

Use `export const dynamic = 'force-dynamic'` and `export const maxDuration = 300`
(5 min — Vercel Pro plan limit for API routes, sufficient for Hamilton LLM calls).

Import `assembleAndRender` from `@/lib/report-engine/assemble-and-render`.
Import `getSql` from `@/lib/crawler-db/connection`.
Import `ReportType` from `@/lib/report-engine/types`.
