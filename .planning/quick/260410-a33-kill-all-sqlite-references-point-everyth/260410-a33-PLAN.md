  <action>
    Create `scripts/audit-data.ts` — a standalone CLI runner that:

    1. Loads `.env` using `dotenv/config` import (already in devDependencies via Next.js)
       so DATABASE_URL is available when run with `npx tsx`.

    2. Imports `getDataQualityReport` from `../src/lib/crawler-db/hygiene`.

    3. Calls `getDataQualityReport()`, awaits the result, then prints a structured
       markdown table to stdout — matching the output format already specified in SKILL.md:

       ```
       | Check                    | Result | Status |
       |--------------------------|--------|--------|
       | Invalid state codes      | N      | PASS/WARN/FAIL |
       | Uncategorized fees       | N      | ... |
       | Null amounts (non-free)  | N      | ... |
       | Stale institutions       | N% of total | ... |
       | Duplicate fee names      | N      | ... |
       | Missing financial data   | N      | ... |
       ```

    4. Status thresholds (match SKILL.md):
       - Invalid state codes: 0 = PASS, >0 = FAIL
       - Uncategorized fees: 0 = PASS, <500 = WARN, >=500 = FAIL
       - Null amounts: <100 = PASS, <1000 = WARN, >=1000 = FAIL
       - Stale institutions: pct of total_institutions — <10% = PASS, <25% = WARN, >=25% = FAIL
       - Duplicate fees: 0 = PASS, >0 = WARN
       - Missing financials: <100 = PASS, <1000 = WARN, >=1000 = FAIL

    5. After the summary table, print:
       - Status distribution (review_status breakdown with counts)
       - Top 10 duplicate fees (if any)
       - Top invalid state codes (if any)

    6. Exit with process.exitCode = 1 if any check is FAIL, 0 otherwise.

    7. In a finally block, call `getSql().end()` to close the Postgres connection
       so the process exits cleanly (without this, the postgres client keeps the
       event loop alive).

    Import pattern:
    ```typescript
    import 'dotenv/config';
    import { getDataQualityReport } from '../src/lib/crawler-db/hygiene';
    import { getSql } from '../src/lib/crawler-db/connection';
    ```

    ```bash
    npx tsx scripts/audit-data.ts
    ```

    Capture and present the full output as part of the task completion. This IS the
    production fee audit — the output is the deliverable, not just a side effect.
