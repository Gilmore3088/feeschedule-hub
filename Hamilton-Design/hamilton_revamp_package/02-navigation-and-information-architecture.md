# 02. Navigation and Information Architecture

## Recommended top navigation

- Home
- Analyze
- Simulate
- Reports
- Monitor
- Admin

## Why this nav wins

Top navigation should map to what the user is trying to do, not to raw data domains.

### Cut or fold
- **Markets**: remove from top nav for now
- **Peers**: remove from top nav for now

Both can live inside:
- filters
- analysis focus
- peer selectors
- report configuration

## Product hierarchy

### Top nav = where am I?
- Home
- Analyze
- Simulate
- Reports
- Monitor
- Admin

### Left rail = what have I worked on?
Use the left rail as workspace memory, not navigation.

Recommended structure:
- New Analysis / New Scenario
- Workspace
  - Saved Analyses
  - Recent Work
- Context
  - Pinned Institutions
  - Peer Sets
- System
  - Settings
  - Support
  - Documentation

### Right-side tabs / pills = what lens am I using?
Use inside Analyze or Monitor only.

Recommended label:
**Analysis Focus**
- Pricing
- Risk
- Peer Position
- Trend

## Naming rules

- Use **Hamilton** as the brand, not the function name of a primary nav tab
- Use **Analyze** for the workspace currently labeled "Hamilton"
- Use **Simulate** as a first-class top-level destination
