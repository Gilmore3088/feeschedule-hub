# 09. Copy and UX Rules

## Copy rules
- lead with implication
- avoid generic AI language
- avoid "assistant is typing" energy
- use brand language sparingly and intentionally
- never let empty states feel blank

## CTA rules
### Home
Primary CTA = Simulate Change

### Analyze
Primary CTA = Simulate a Change
Secondary = Show Peer Distribution / View Risk Drivers

### Simulate
Primary CTA = Generate Board Scenario Summary

### Report
Primary CTA = Export PDF

### Monitor
Primary CTA = Review Pricing or Run Scenario

## Screen boundary rules
- Only Simulate recommends a position
- Report repeats the recommendation cleanly
- Analyze explores, it does not decide
- Monitor surfaces signals, it does not recreate analysis

## Labeling rules
Use these labels consistently:
- Hamilton's View
- What Changed
- What This Means
- Why It Matters
- Recommended Position
- Priority Alert
- Signal Feed
- Analysis Focus
