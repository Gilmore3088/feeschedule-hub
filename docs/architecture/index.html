2. Agent Team (Who Actually Runs)

There is no central orchestrator. Modal cron triggers four autonomous agent loops; they communicate through the agent_messages table, not with each other. Hamilton is a user-facing synthesis tool, not a scheduled loop. "Atlas" is a permission label in DB auth checks — no process, no code.

3. The Adversarial Handshake

The SQL gate promote_to_tier3 requires BOTH a darwin-accept AND a knox-accept message in agent_messages before a fee can be published. publish-fees posts darwin's accept; Knox posts knox's. Either can reject and block publication.

sequenceDiagram autonumber participant Crawl as Crawler participant FR as fees_raw participant Darwin as Darwin participant FV as fees_verified participant Knox as Knox participant AM as agent_messages participant SQL as promote_to_tier3() participant FP as fees_published Crawl->>FR: insert (source, amount, fee_name) Darwin->>FR: fetch unverified Darwin->>FV: promote_to_tier2 (canonical_fee_key, confidence) Darwin->>AM: insert (darwin, intent=accept, fee_verified_id=N) Knox->>FV: fetch rows with no knox message alt peer_median check passes AND waiver rule passes Knox->>AM: insert (knox, intent=accept, fee_verified_id=N) else violates a rule Knox->>AM: insert (knox, intent=reject, reasons=[…]) end Note over SQL,AM: both darwin-accept AND knox-accept required SQL->>AM: check (darwin_accept AND knox_accept) alt both accepted SQL->>FP: INSERT fee_published else either missing / knox rejected SQL-->>FP: RAISE EXCEPTION (stays in fees_verified) end

6. Institution Identity Map

Every fee, crawl run, and analysis row joins through one internal canonical key: crawl_targets.id. Federal datasets each key off a different external identifier — we store all of them as side-columns on the same row so any source can join cleanly. Without this, matching a published fee back to a Call Report or a Fed supervisory record relies on fragile name-matching.

9. Security Boundaries & Auth Flow

Who can touch what. Agent writes pass through a gateway that authenticates, enforces per-tool role checks, records every call to agent_auth_log, and checks agent_budgets before dispatch. User writes pass through Next.js server actions with session cookies.
